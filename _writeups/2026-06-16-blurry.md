## Reconnaissance

The assessment began with a port scan to identify exposed services. It revealed two open ports: port 80 running HTTP and port 21 for SSH. Browsing the web service on port 80 redirected to a specific subdomain, which was added to the local hosts file to ensure proper name resolution during further testing.

![Starting with a port scan against the target to identify exposed services](/assets/images/blurry/94-1.png)
*Starting with a port scan against the target to identify exposed services*

## Enumeration / Analysis

Exploring the web application surfaced a feature that allowed the creation of new credentials. Submitting an arbitrary username granted immediate access as a developer user. That access level was enough to interact with the ClearML platform, which required local configuration using the credentials provided by the application.

![While exploring the web application, discovering a functionality that allowed the creation of new credentials](/assets/images/blurry/94-2.png)
*While exploring the web application, discovering a functionality that allowed the creation of new credentials*

ClearML is an MLOps solution used to manage and run machine learning models. After installing and initializing ClearML locally, it was configured with the details obtained from the application. This step surfaced an issue caused by missing subdomains, which had to be added so the client could communicate correctly with the ClearML services.

![ClearML is an MLOps solution used to manage and execute machine learning models](/assets/images/blurry/94-3.png)
*ClearML is an MLOps solution used to manage and execute machine learning models*

Research into known vulnerabilities affecting ClearML pointed to a relevant issue: **CVE-2024-24590: Pickle Load on Artifact Get**. It allows arbitrary code execution through a malicious Pickle file when an artifact is downloaded. Understanding how an administrator retrieves and opens artifacts was key, since that action is what triggers the payload.

![While researching potential vulnerabilities affecting ClearML, identifying a relevant issue: CVE-2024-24590: Pickle Load on Artifact Get](/assets/images/blurry/94-4.png)
*While researching potential vulnerabilities affecting ClearML, identifying a relevant issue: CVE-2024-24590: Pickle Load on Artifact Get*

## Exploitation

Additional technical details from public research confirmed that this vulnerability could be abused to craft a malicious artifact containing a reverse shell payload. A detailed explanation of this attack vector can be found in the following research article: [https://hiddenlayer.com/research/not-so-clear-how-mlops-solutions-can-muddy-the-waters-of-your-supply-chain/](https://hiddenlayer.com/research/not-so-clear-how-mlops-solutions-can-muddy-the-waters-of-your-supply-chain/?source=post_page-----b44bc2c1aea0--------------------------------). After the malicious artifact was uploaded to the project, the exploit ran and returned a reverse shell as a low-privileged user.

![Public research describing the malicious-artifact attack](/assets/images/blurry/94-5.png)
*Public research describing the malicious-artifact attack*

## Privilege Escalation

To map privilege-escalation paths, the commands available with elevated privileges were checked.

```bash
sudo -l
```

![To identify possible privilege escalation paths, checking which commands could be executed with elevated privileges](/assets/images/blurry/94-6.png)
*To identify possible privilege escalation paths, checking which commands could be executed with elevated privileges*

The output revealed a script that searches for files ending in `.pth` inside the `models` directory, strips malicious code, and then executes them. Crucially, the current user has write permissions on the `models` directory, which makes this a viable escalation vector.

![The sudo script that processes .pth files in the models directory](/assets/images/blurry/94-7.png)
*The sudo script that processes .pth files in the models directory*

Abusing this behavior required the `torch` library to generate a compatible model file. Once it was installed, a malicious model file was created and uploaded to the writable directory on the target.

![Installing torch to build a compatible malicious model file](/assets/images/blurry/94-8.png)
*Installing torch to build a compatible malicious model file*

Finally, the allowed sudo command was executed, which processed and ran the malicious model file.

![Finally, executing the allowed sudo command, which processed and ran the malicious model file](/assets/images/blurry/94-9.png)
*Finally, executing the allowed sudo command, which processed and ran the malicious model file*

This returned a reverse shell with root privileges, completing the compromise.

![Reverse shell obtained with root privileges](/assets/images/blurry/94-10.png)
*Reverse shell obtained with root privileges*
