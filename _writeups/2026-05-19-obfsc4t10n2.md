## Initial Triage

The challenge provides an `.xls` file presented as part of a phishing campaign.

![the provided XLS file](/assets/images/obfsc4t10n2/Pasted-image-20260519184954.png)
*Initial view of the provided XLS file*

The file claims to contain encrypted content that requires the user to enable macros to view it — a common social engineering pretext used to lure victims into enabling malicious macro execution.

![document presenting an "encrypted" content lure](/assets/images/obfsc4t10n2/Pasted-image-20260519185204.png)
*Lure shown to the victim, prompting macro execution*

Since this is a phishing campaign delivered as an `.xls` file, a hidden macro is suspected. The container type is identified first.

```bash
file oBfsC4t10n2.xls
```

![file command output identifying the container format](/assets/images/obfsc4t10n2/Pasted-image-20260519190123.png)
*File type identified as a legacy OLE2 (XLS) document*

## Static Analysis

As the file is a legacy OLE2 document, the **Didier Stevens Suite** is used for analysis. `oledump.py` lists the streams contained within the OLE structure.

```bash
python oledump.py oBfsC4t10n2.xls
```

![oledump output listing the OLE streams](/assets/images/obfsc4t10n2/Pasted-image-20260519191223.png)
*OLE streams contained within the document*

Nothing immediately stands out from the stream listing, so analysis continues using **oletools**, a suite specifically designed for VBA/XLM macro analysis.

### Initial Triage with oleid

`oleid` performs a quick triage of the document, checking for indicators such as macros, encryption, and embedded objects.

```bash
oleid oBfsC4t10n2.xls
```

This identifies the presence of an **XLM macro** — an older macro format (also known as Excel 4.0 macros) that predates VBA and is often used to evade detection, since many analysis tools focus primarily on VBA.

![oleid confirming the presence of an XLM macro](/assets/images/obfsc4t10n2/Pasted-image-20260519191706.png)
*oleid output confirming an XLM macro is present*

### Extracting and Deobfuscating the XLM Macro

`olevba` is used to extract and deobfuscate the XLM macro formulas.

```bash
olevba oBfsC4t10n2.xls
```

> XLM (Excel 4.0) macros store their logic directly inside cell formulas rather than in a separate VBA module. This makes them harder to spot during a casual review of the workbook, while still allowing the same level of scripting capability as VBA, including file downloads and process execution.

The deobfuscated macro shows that it downloads a `.dll` file and subsequently executes it — a typical pattern for delivering a second-stage payload.

![macro formulas showing a DLL download and execution](/assets/images/obfsc4t10n2/Pasted-image-20260519192417.png)
*Deobfuscated macro revealing the DLL download and execution routine*

## Flag Recovery

Reviewing the macro formulas further reveals the beginning of the flag, embedded as part of the obfuscated formula chain.

![macro formulas revealing the start of the flag](/assets/images/obfsc4t10n2/Pasted-image-20260519224705.png)
*Portion of the flag visible within the macro formulas*

The full flag is reconstructed by manually concatenating the fragments scattered across the macro formulas.
