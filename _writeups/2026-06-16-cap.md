## Port Scanning and Enumeration

A full port scan was run against the target to identify the exposed services.

```bash
nmap -Pn -sV -sC -p- --open 10.10.10.245 -oN scan.txt
```

The results show HTTP, FTP, and SSH running on the host. To get clean name resolution, the target IP is added to the local hosts file.

```bash
nano /etc/hosts
```

## Dashboard Analysis

Browsing to the host presents a dashboard interface. It exposes a **Download** button, so the request behind it is intercepted with Burp Suite to see how it behaves.

The response itself contains nothing directly useful, but it returns a `.pcap` file that can be examined in Wireshark.

![PCAP file download](/assets/images/cap/24-1.png)

## Traffic Analysis

Opening the capture in Wireshark and following the traffic reveals cleartext credentials belonging to a valid user.

![Credentials in Wireshark](/assets/images/cap/24-2.png)
*Cleartext credentials recovered from the capture*

## Initial Access

Those credentials are used to log in over SSH as **nathan**, which gives access to the `user.txt` flag.

To map out privilege-escalation paths, **LinPEAS** is uploaded and run on the target, with the output saved for review.

```bash
./linpeas.sh > linpeas.txt
```

![LinPEAS output](/assets/images/cap/24-3.png)
*LinPEAS results on the target*

## Privilege Escalation

The LinPEAS output flags a misconfigured **bind** binary that carries elevated capabilities and can be abused for escalation. A matching technique on **GTFOBins** provides a command that drops to a root shell.

```python
python3.8 -c 'import os; os.setuid(0); os.system("/bin/bash")'
```

Running it spawns a root shell with full system access and exposes the `root.txt` flag.

![Root flag](/assets/images/cap/24-4.png)
