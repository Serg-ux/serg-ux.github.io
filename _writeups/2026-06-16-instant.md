## Reconnaissance

The engagement began with a full TCP port scan to identify exposed services and gather banner information.

```bash
nmap -sVC -p- --open -sS --min-rate 5000 -v -n --stats-every=5s -Pn -oN instant_scan 10.10.11.37
```

Based on the scan results, the application relied on a specific domain name, which needed to be added to the local `/etc/hosts` file to ensure proper resolution.

![Adding the application domain to /etc/hosts](/assets/images/instant/183-1.png)
*Adding the application domain to /etc/hosts*

Enumeration of subdomains and directories did not reveal any immediately interesting attack vectors.

## Enumeration / Analysis

Exploring the main web application revealed that it offered an Android application package (`.apk`) for download. Since mobile apps often embed hardcoded secrets or endpoints, the file was downloaded for static analysis.

The APK was decompiled with `jadx-gui` to review its source code and search for sensitive information.

![Decompiling the APK with jadx-gui to inspect its source](/assets/images/instant/183-2.png)
*Decompiling the APK with jadx-gui to inspect its source*

The `AdminActivities` class contained a hardcoded JSON Web Token (JWT) tied to administrative functionality.

![Within the class, discovering a hardcoded JSON Web Token (JWT) associated with administrative functionality](/assets/images/instant/183-3.png)
*Within the class, discovering a hardcoded JSON Web Token (JWT) associated with administrative functionality*

The APK analysis also revealed two additional subdomains referenced by the application.

![The APK analysis also revealed two additional subdomains referenced by the application](/assets/images/instant/183-4.png)
*The APK analysis also revealed two additional subdomains referenced by the application*

The first subdomain did not expose any interesting functionality. However, the second one, `swagger-ui.instant.htb`, exposed a Swagger interface where application logs could be viewed.

A GET request to this endpoint was intercepted in Burp Suite to inspect both the request structure and the server response.

Supplying the previously discovered admin JWT exposed a Local File Inclusion vulnerability that allowed arbitrary file reads on the server.

![Local File Inclusion confirmed with the admin JWT](/assets/images/instant/183-5.png)
*Local File Inclusion confirmed with the admin JWT*

![Reading arbitrary files through the LFI](/assets/images/instant/183-6.png)
*Reading arbitrary files through the LFI*

This vulnerability was used to enumerate system users, which identified a valid account named `shirohige`.

The LFI was then leveraged to read this user's private SSH key directly from the filesystem.

![Leveraging the LFI to read the private SSH key of this user directly from the filesystem](/assets/images/instant/183-7.png)
*Leveraging the LFI to read the private SSH key of this user directly from the filesystem*

![Leveraging the LFI to read the private SSH key of this user directly from the filesystem](/assets/images/instant/183-8.png)
*Leveraging the LFI to read the private SSH key of this user directly from the filesystem*

## Exploitation

The private key was saved locally and used to authenticate to the target over SSH as `shirohige`.

![Authenticating over SSH as shirohige with the recovered private key](/assets/images/instant/183-9.png)
*Authenticating over SSH as shirohige with the recovered private key*

![Authenticating over SSH as shirohige with the recovered private key](/assets/images/instant/183-10.png)
*Authenticating over SSH as shirohige with the recovered private key*

![Authenticating over SSH as shirohige with the recovered private key](/assets/images/instant/183-11.png)
*Authenticating over SSH as shirohige with the recovered private key*

Once logged in, local enumeration of the filesystem began. It surfaced a directory named `backups` that appeared to hold archived data.

![Once logged in, beginning local enumeration of the filesystem](/assets/images/instant/183-12.png)
*Once logged in, beginning local enumeration of the filesystem*

Inside this directory was a backup file related to application sessions.

![Inside this directory, finding a backup file related to application sessions](/assets/images/instant/183-13.png)
*Inside this directory, finding a backup file related to application sessions*

## Privilege Escalation

To analyze it further, the session backup file was transferred to the attacking machine using a temporary Python HTTP server and `wget`.

![Transferring the session backup to the attacking machine](/assets/images/instant/183-14.png)
*Transferring the session backup to the attacking machine*

![Processing the recovered session backup](/assets/images/instant/183-15.png)
*Processing the recovered session backup*

Processing and cracking the extracted data recovered credentials belonging to the `root` user.

![After processing and cracking the extracted data, recovering credentials belonging to the user](/assets/images/instant/183-16.png)
*After processing and cracking the extracted data, recovering credentials belonging to the user*

With valid root credentials obtained, full administrative access to the system was achieved, completing the compromise.
