## Reconnaissance

A port scan of the target identifies that **SSH** and **HTTP** services are running. Since SSH offers a limited attack surface without credentials, the focus shifts to the HTTP service.

The target's IP address is added to `/etc/hosts` for easier access.

## Enumeration

Reviewing the website's source code does not reveal anything of immediate interest. Burp Suite is used to intercept traffic and analyze the response from a `GET` request to the site.

As an alternative, an `OPTIONS` request is sent with `curl` to inspect the response headers directly.

```bash
curl -X OPTIONS http://10.10.10.242 -i
```

The HTTP headers reveal that the web server is running **PHP 8.1.0-dev**, which is significant for exploitation.

## Exploitation

PHP 8.1.0-dev was distributed with a backdoor inserted into a compromised git repository mirror: requests containing a specific header are passed directly to PHP's `eval()`, resulting in unauthenticated remote code execution. A public exploit for this is located using SearchSploit.

![SearchSploit results for PHP 8.1.0-dev RCE](/assets/images/knife/Pasted-image-20260103182146.png)

The exploit is copied to a local working directory for use.

![exploit copied to the local working directory](/assets/images/knife/Pasted-image-20260103182150.png)
*Exploit copied locally before execution*

Executing the exploit against the web application's backdoor provides access to the target system as the user **james**.

![shell obtained as the user james](/assets/images/knife/Pasted-image-20260103182208.png)
*Shell access obtained as the user james*

## Post-Exploitation — User Flag

As directory navigation is restricted, the user flag is retrieved with a direct command.

```bash
cat /home/james/user.txt
```

## Privilege Escalation

Running `sudo -l` reveals that the current user can execute the **knife** command as root without a password.

`knife` is the command-line tool for **Chef**, a configuration management platform. It supports an `exec` subcommand that runs arbitrary Ruby code — including the ability to invoke system commands.

Since the current shell does not have a TTY, a PHP-based reverse shell is executed first to obtain a fully interactive session before escalating privileges.

The root flag is then read by abusing `knife exec` to run a Ruby `system()` call as root.

```bash
sudo knife exec -E 'system("cat /root/root.txt")'
```

> A passwordless `sudo` rule for `knife exec` is equivalent to a passwordless `sudo bash`, since `knife exec` can execute arbitrary Ruby code — including direct shell command execution via `system()` — as root.
