## Reconnaissance

The assessment began with a port scan to identify the exposed services. The host is identified as a Domain Controller.

![Port scan results](/assets/images/forest/200-1.png)
*Port scan identifying the host as a Domain Controller*

Given those services, LDAP enumeration is run against the htb.local domain to gather directory information.

```bash
ldapsearch -x -H 'ldap://10.10.10.161' -b "dc=htb,dc=local"
```

A Domain Controller handles authentication, authorization, and directory services within an Active Directory environment, so its LDAP data is a rich source of accounts. Filtering the output exposes several service accounts.

```bash
ldapsearch -x -H 'ldap://10.10.10.161' -b "dc=htb,dc=local" | grep "svc-"
```

![Service accounts in LDAP output](/assets/images/forest/200-2.png)
*Service accounts revealed by the LDAP query*

One of them is a service account used to run platform-related services.

## Enumeration / Analysis

A Ticket Granting Ticket (TGT) is requested for that service account to retrieve its Kerberos hash (an AS-REP roasting condition).

![TGT request for the service account](/assets/images/forest/200-3.png)
*Requesting a TGT to obtain the account's Kerberos hash*

The hash is then cracked offline with John the Ripper.

![Hash cracked with John the Ripper](/assets/images/forest/200-4.png)
*Cracking the recovered hash offline*

With valid credentials in hand, port 5985 is open, meaning WinRM is available for remote sessions. WinRM is used to connect as the alfresco user, which gives access to the user flag.

![WinRM session as alfresco](/assets/images/forest/200-5.png)
*Remote WinRM session established as alfresco*

## Privilege Escalation

To map escalation paths in the domain, BloodHound collects and visualizes the Active Directory relationships.

```bash
bloodhound-python -d htb.local -u 'svc-alfresco' -p 's3rvice' -gc forest.htb.local -c all --zip -ns 10.10.10.161
```

![BloodHound collection](/assets/images/forest/200-6.png)
*Collecting AD data with BloodHound*

The GUI shows that alfresco belongs to several groups.

![alfresco group membership](/assets/images/forest/200-7.png)
*Group memberships of the alfresco user*

One path shows that the Exchange Windows Permissions group holds WriteDACL over the domain, which allows modifying the access control lists on domain objects.

![WriteDACL over the domain](/assets/images/forest/200-8.png)
*The Exchange Windows Permissions group has WriteDACL on the domain*

A new user is created and added to both the Exchange Windows Permissions group and the Remote Management Users group to extend its privileges.

![New user added to privileged groups](/assets/images/forest/200-9.png)
*Adding a controlled user to the privileged groups*

The changes apply successfully.

![Changes applied](/assets/images/forest/200-10.png)
*Group changes committed*

PowerView is then downloaded and imported into the current PowerShell session, after running Bypass-4MSI to evade Defender. Using Add-ObjectACL with the new credentials grants DCSync rights to the controlled user.

![DCSync rights granted via Add-ObjectACL](/assets/images/forest/200-11.png)
*Granting DCSync rights with Add-ObjectACL*

DCSync abuses directory-replication permissions to pull password hashes straight from the Domain Controller.

![DCSync replication abuse](/assets/images/forest/200-12.png)
*DCSync replicates secrets directly from the DC*

The controlled user now has DCSync permissions, so the domain hashes can be extracted from the DC.

![Extracting domain hashes](/assets/images/forest/200-13.png)
*Dumping domain user hashes via DCSync*

![Extracted hashes](/assets/images/forest/200-14.png)
*Recovered NTLM hashes for domain accounts*

The Administrator NTLM hash is then reused in a Pass-the-Hash attack with crackmapexec.

![Pass-the-Hash with the Administrator hash](/assets/images/forest/200-15.png)
*Authenticating as Administrator via Pass-the-Hash*

crackmapexec can also run commands with that hash, giving direct access to the root flag.

```bash
crackmapexec smb <IP> -u 'Administrator' -H 'NTML_HASH' -d 'htb.local' -x "type C:\Users\Administrator\Desktop\root.txt"
```
