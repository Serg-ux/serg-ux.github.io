## Initial Enumeration

A full port scan was run against the target to identify the exposed services.

```bash
nmap 10.10.10.37 -Pn -sC -sV -p- --open -T4
```

The scan reveals three services: **FTP**, **SSH**, and **HTTP**. The target IP is added to `/etc/hosts` for hostname resolution. To fingerprint the web service, **WhatWeb** is run, which shows the server running **Apache** with **WordPress**.

*WhatWeb fingerprint of the web service*

![WordPress detected](/assets/images/blocky/81-2.png)
*WordPress identified on the host*

## Credential Discovery

The `/plugins` directory under `blocky.htb` contains two files that appear to hold database connection details. Decompiling the `.jar` with **JD-GUI** recovers hardcoded credentials.

![Decompiled JAR credentials](/assets/images/blocky/81-7.png)
*Credentials recovered from the decompiled JAR*

Those credentials log in to **phpMyAdmin** and grant access to the database dashboard.

![phpMyAdmin access](/assets/images/blocky/81-8.png)
*Authenticated phpMyAdmin dashboard*

**WPScan** enumerates additional directories and possible vectors.

The phpMyAdmin panel exposes credentials for the **Notch** user.

![Notch credentials](/assets/images/blocky/81-11.png)

After resetting the password, the WordPress admin panel becomes accessible.

WPScan also flags the **TwentySeventeen** theme as a vector. Since the theme runs PHP files, a PHP reverse shell based on the **pentestmonkey** payload is uploaded through it.

![Reverse shell via theme](/assets/images/blocky/81-14.png)
*Reverse shell planted in the theme*

Executing the file returns a reverse shell and access to the `user.txt` flag.

![User flag](/assets/images/blocky/81-15.png)
*User flag obtained*

## Privilege Escalation

The limited shell cannot switch directly to the **Notch** user. However, the root password recovered earlier from the decompiled JAR is reused to authenticate over **SSH**.

![SSH access](/assets/images/blocky/81-16.png)
*SSH session using the reused password*

`sudo -l` then shows what the user can run as root.

![sudo privileges](/assets/images/blocky/81-17.png)
*sudo rights for the current user*

The output allows every command without a password, so `sudo su` yields a full root shell and the `root.txt` flag.

![Root flag](/assets/images/blocky/81-18.png)
*Root flag obtained*
