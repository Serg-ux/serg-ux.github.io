## Reconnaissance

The assessment began with a port scan to identify exposed services. The results pointed to a web application as the main attack surface, so the effort focused on HTTP enumeration.

![Port scan results for the target](/assets/images/heal/182-1.png)
*Port scan results for the target*

## Enumeration / Analysis

The next step was subdomain enumeration to find additional virtual hosts that might expose hidden functionality, using ffuf with a common wordlist to fuzz the Host header against the main domain.

```bash
ffuf -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -u http://heal.htb -H "Host: FUZZ.heal.htb" -fc 404 -mc 200
```

These credentials did not work over SSH, but they were valid for the LimeSurvey application, granting access to Ralph’s administrative panel.

## Exploitation

To exploit the vulnerability, a malicious plugin was prepared by zipping a `reverse.php` file together with a compatible `config.xml` from version 6.0. The plugin was uploaded through the LimeSurvey interface and activated.

![Malicious plugin packaged for upload to LimeSurvey](/assets/images/heal/182-21.png)
*Malicious plugin packaged for upload to LimeSurvey*

After activation, the reverse shell was triggered by visiting the plugin through its URL, returning a shell as the `www-data` user.

Enumerating the system surfaced database credentials inside the LimeSurvey configuration file located at:

```bash
/var/www/limesurvey/application/config/config.php
```

![While enumerating the system, finding database credentials inside the LimeSurvey configuration file located at:](/assets/images/heal/182-22.png)
*While enumerating the system, finding database credentials inside the LimeSurvey configuration file located at:*

The credentials belonged to the user `ron` and allowed authentication over SSH.

```bash
ron:AdmiDi0_pA$$w0rd
```

![The recovered credentials for the user ron, valid over SSH](/assets/images/heal/182-23.png)
*The recovered credentials for the user ron, valid over SSH*

## Privilege Escalation

Once logged in as `ron`, linpeas was uploaded to identify potential privilege-escalation vectors.

![Once logged in as Ron, uploading linpeas to identify potential privilege escalation vectors on the system](/assets/images/heal/182-24.png)
*Once logged in as Ron, uploading linpeas to identify potential privilege escalation vectors on the system*

Enumeration revealed a local service listening on port 8500. To reach it, an SSH port-forwarding tunnel was created.

![During enumeration, discovering a local service listening on port 8500](/assets/images/heal/182-25.png)
