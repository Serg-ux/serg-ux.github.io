## Reconnaissance

A scan for open TCP ports is performed first.

```bash
nmap -p- --open -Pn 10.129.60.85
```

The same is done for UDP ports.

```bash
nmap -sU --open -Pn -vv 10.129.60.85
```

Three services are identified.

![three open services identified](/assets/images/paper/Pasted-image-20260128191347.png)
*Services identified by the initial scans*

A version and script scan is then run for more detail.

```bash
nmap -sC -sV -Pn -p 22,80,443 --open -T4 10.129.60.85 --stats-every=5s -oN scan.txt
```

![version and script scan results](/assets/images/paper/Pasted-image-20260128191556.png)
*Detailed service information for SSH and the two web ports*

SSH and two web services are present. The focus shifts to the web services to find vulnerabilities.

## Enumeration

Directory enumeration is performed against the main web service.

```bash
gobuster dir -u http://10.129.60.85/ -w /usr/share/wordlists/dirb/big.txt
```

![gobuster directory enumeration results](/assets/images/paper/Pasted-image-20260128192457.png)
*Directory brute-force results against the main webpage*

Nothing of immediate interest is found through brute-forcing, so Burp Suite is used to inspect how the application behaves. A response header reveals important information: `X-Backend-Server: office.paper`.

`X-Backend-Server` is an HTTP response header that indicates which backend server actually handled the request, typically added by a reverse proxy — revealing an internal hostname that is not normally exposed.

This domain is added to `/etc/hosts`. Accessing it reveals a **WordPress** site with comments.

![WordPress site with comments on office.paper](/assets/images/paper/Pasted-image-20260128194440.png)
*WordPress instance discovered on the office.paper vhost*

Reviewing the comments shows that a user named "nick" instructed others to remove drafts because they are not secure.

![comment referencing insecure drafts](/assets/images/paper/Pasted-image-20260128195254.png)
*Comment hinting that draft posts are not properly protected*

Enumerating directories on this site reveals paths typical of a WordPress installation. WPScan is used to fingerprint the application further.

```bash
wpscan --url http://office.paper
```

The WordPress version is identified, which can also be confirmed via Wappalyzer.

![WordPress version identified by WPScan](/assets/images/paper/Pasted-image-20260128195424.png)
*WordPress version returned by WPScan*

![WordPress version confirmed via Wappalyzer](/assets/images/paper/Pasted-image-20260128195445.png)

## Exploitation — CVE-2019-17671

This WordPress version is affected by [CVE-2019-17671](https://nvd.nist.gov/vuln/detail/CVE-2019-17671), a vulnerability that allows unauthenticated users to read private and draft posts by directly accessing their REST API endpoint, bypassing the visibility restrictions normally enforced for non-logged-in users.

![exploit reference for CVE-2019-17671](/assets/images/paper/Pasted-image-20260128195932.png)
*Reference to the CVE-2019-17671 exploit*

Using this vulnerability, the contents of the previously hinted-at private drafts are read.

![private draft contents disclosed via CVE-2019-17671](/assets/images/paper/Pasted-image-20260128195954.png)
*Secret information disclosed through the private post*

The disclosed content references an additional subdomain, which is added to `/etc/hosts`. This subdomain hosts a login panel. Following a registration link embedded in the leaked content allows a new user account to be created.

![user registration on the new subdomain](/assets/images/paper/Pasted-image-20260128200422.png)
*Registration page reached via the leaked invite link*

## Credential Discovery — Hubot Environment Variables

In the bot's working directory (`hubot`), an environment file is present that contains credentials for the chat platform's service account. These variables can be retrieved either by reading the process environment:

```bash
cat /proc/self/environ
```

or by reading the environment file directly, which contains entries such as:

```bash
export ROCKETCHAT_URL='http://127.0.0.1:48320'
export ROCKETCHAT_USER=recyclops
export ROCKETCHAT_PASSWORD=Queenofblad3s!23
export ROCKETCHAT_USESSL=false
export RESPOND_TO_DM=true
export RESPOND_TO_EDITED=true
export PORT=8000
export BIND_ADDRESS=127.0.0.1
```

> Environment files used to configure chat bots and similar integrations frequently contain service-account credentials for the platforms they connect to. Recovering this file via an LFI provides a direct path to credentials that may be reused for other access (such as SSH) on the host.

An SSH attempt with these credentials is denied. Enumerating `/etc/passwd` to identify valid local users shows that the two most relevant accounts are `dwight` and `rocketchat`.

## Privilege Escalation — CVE-2021-3560

Running `linpeas` reveals that the system is vulnerable to [CVE-2021-3560](https://nvd.nist.gov/vuln/detail/cve-2021-3560), a vulnerability in **Polkit**'s `pkexec` D-Bus authentication helper. By exploiting a race condition while a D-Bus connection is being dropped, an unprivileged user can create a new account with administrative (sudo) privileges.

> CVE-2021-3560 allows a local unprivileged user to perform privileged actions through `polkitd` by sending a D-Bus request and disconnecting before authentication completes, causing Polkit to process the request as if it came from a privileged context.

A new user with sudo privileges is created by exploiting this vulnerability.

![new sudo-privileged user created via CVE-2021-3560](/assets/images/paper/Pasted-image-20260128213452.png)
*Privileged user account created through the Polkit vulnerability*
