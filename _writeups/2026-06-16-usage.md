## Reconnaissance

A full TCP port scan was run to identify the exposed services.

```bash
nmap -sV -sC -p- --open 10.10.11.18 -oN scan.txt
```

The scan shows SSH and HTTP open.

## Enumeration

The web application has a login page. A button in the top-right corner redirects to the subdomain **admin.usage.htb**, so both the main domain and the admin subdomain are added to `/etc/hosts`. The admin subdomain serves an administrative login page, and standard credentials do not work.

The `/forget-password` endpoint returns a "not found" message when an unknown email is submitted, which hints that the backend is building SQL queries from the input. An initial sqlmap run returns nothing, but a second run with adjusted parameters confirms a SQL injection and extracts a password hash.

```bash
hashcat -m 3200 hash /usr/share/wordlists/rockyou.txt
```

The hash cracks to valid credentials, **admin:whatever1**, which grant access to `http://admin.usage.htb/`.

## Exploitation

The only file-upload feature in the admin panel is the profile-picture update. Its validation is weak enough that a PHP file disguised with a `.jpg` extension can be uploaded, matching **CVE-2023-24249**.

![Weak upload validation](/assets/images/usage/39-1.png)
*The profile-picture upload accepts a disguised PHP file*

The file is first uploaded as `.jpg` and the request is intercepted in Burp Suite, where the extension is changed from `.jpg` to `.php` to bypass the type check.

![Extension changed in Burp](/assets/images/usage/39-2.png)
*Renaming the upload to .php during interception*

The upload path is recovered through the download feature, and requesting the PHP file returns a reverse shell.

![Reverse shell obtained](/assets/images/usage/39-3.png)
*Reverse shell from the uploaded PHP file*

## Post-Exploitation Analysis

The system has a user named **xander**, and the goal is to pivot to that account and then to root. The home directory of the user **dash** holds several hidden files, one of which, `.monitrc`, contains a plaintext password tied to a local account. Those credentials grant SSH access as **xander**.

![SSH access as xander](/assets/images/usage/39-4.png)
*SSH session established as xander*

## Privilege Escalation

`sudo -l` is run as **xander** to look for escalation paths.

![sudo privileges for xander](/assets/images/usage/39-5.png)
*sudo rights available to xander*

The output shows that the `usage_management` script can be run with elevated privileges. Inspecting it reveals that it uses `7zip` to archive files from `/var/www/html` into `/var/backups`. Working from `/var/www/html` and abusing 7zip's wildcard handling makes it possible to influence the archive operation, which leads to command execution as root.

![Wildcard abuse yields root](/assets/images/usage/39-6.png)
*7zip wildcard abuse leading to root command execution*
