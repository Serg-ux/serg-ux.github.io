## Port Scanning

Scanning all ports to identify available services:

```bash
nmap -sVC -p- --open -sS --min-rate 5000 -v -n -Pn -oN lantern_scan 10.10.11.29
```

The scan redirects to a web page:

![Lantern landing page](/assets/images/lantern/122-1.png)
*Lantern landing page*

The domain is added to **/etc/hosts**:

```bash
nano /etc/hosts
```

## Initial Web Recon

At first glance, nothing interesting is visible. Subdomain enumeration fails. The server uses Blazor (.NET), so it may have DLL files.

A POST request exists at **/vacancies**, possibly vulnerable to SSRF. It only accepts PDFs.

![Vacancies POST request](/assets/images/lantern/122-2.png)
*Vacancies POST request*

## SSRF Exploitation

The server is vulnerable via **X-Skipper-Proxy**:

[X-Skipper-Proxy v0.13.237 - Server Side Request Forgery (SSRF)](https://www.exploit-db.com/exploits/51111)

This allows reading local files on the server.

![Local file reading via SSRF](/assets/images/lantern/122-3.png)
*Local file reading via SSRF*

Using Burp Suite, the GET request is intercepted and modified to perform SSRF.

![SSRF request interception](/assets/images/lantern/122-4.png)
*SSRF request interception*

![SSRF response](/assets/images/lantern/122-5.png)
*SSRF response*

## Port Discovery via SSRF

FFUF is used to enumerate internal ports:

```bash
ffuf -u http://lantern.htb/ -H "X-Skipper-Proxy: http://localhost:FUZZ" -w /usr/share/seclists/Discovery/Infrastructure/common-http-ports.txt
```

![FFUF internal port discovery](/assets/images/lantern/122-6.png)

## File Upload & User SSH Key Extraction

Investigating the file uploads reveals a vulnerable DLL module (FileUpload.dll) that allows writing files to /opt/components.

![FileUpload.dll vulnerability](/assets/images/lantern/122-26.png)
*FileUpload.dll vulnerability*

A DLL is created and modified to read **tomas' id_rsa**:

```bash
str1 = new StreamReader((Stream) new MemoryStream(File.ReadAllBytes("/home/tomas/.ssh/id_rsa")), Encoding.UTF8, true).ReadToEnd();
```

Uploading it and intercepting the request returns the SSH key. Connecting via SSH:

## Privilege Escalation

Running **sudo -l** shows a command that can be executed as root:

![sudo privileges](/assets/images/lantern/122-30.png)
*sudo privileges*

Using **ps aux | grep root**, the running root processes are identified:

![Root processes](/assets/images/lantern/122-31.png)
*Root processes*

One process runs **automation.sh**. Monitoring it with **procmon**:

![Procmon capture](/assets/images/lantern/122-32.png)
*Procmon capture*

![Procmon running](/assets/images/lantern/122-33.png)
*Procmon running*

After stopping the capture (CTRL+C), a SQLite database containing the root password is found:

![SQLite database](/assets/images/lantern/122-35.png)
*SQLite database*

![Transfer database locally](/assets/images/lantern/122-36.png)
*Transfer database locally*

Using this password, logging in as root grants access to **root.txt**.
