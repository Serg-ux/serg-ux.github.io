## Port Scanning

A full port scan was run to discover the services in use.

```bash
nmap -sVC -p- --open -sS --min-rate 5000 -v -n --stats-every=5s -Pn -oN mirai_scan <IP>
```

The scan returns several services:

- 22 ssh OpenSSH 6.7p1 Debian 5+deb8u3 (protocol 2.0)
- 53 domain dnsmasq 2.76
- 80 http lighttpd 1.4.35
- 1456 upnp Platinum UPnP 1.0.5.13
- 32400 http Plex Media Server httpd
- 32469 upnp Platinum UPnP 1.0.5.13

## HTTP Service Investigation

Port 80 returns a blank page, but inspecting the response in Burp Suite reveals the **X-Pi-Hole** header. Pi-hole is a [DNS sinkhole](https://en.wikipedia.org/wiki/DNS_Sinkhole) that filters unwanted content without any client-side software.

Pi-hole serves its admin dashboard at **/admin**, which accepts the default credentials:

```bash
pi:raspberry
```

The same credentials work over SSH and return a shell as the user **pi**.

![User shell access](/assets/images/mirai/135-5.png)
*SSH shell as the pi user*

## USB Forensics

`lsblk` shows the attached block devices, including a USB stick mounted at **/media/usbstick**.

![lsblk output](/assets/images/mirai/135-7.png)
*A USB stick mounted at /media/usbstick*

The USB stick holds an archive worth examining forensically.

![Important archive on USB stick](/assets/images/mirai/135-8.png)
*Contents of the USB stick*

A forensic image of the stick is created with **dcfldd**, recording hashes to preserve integrity:

```bash
dcfldd if=/dev/sdb of=/tmp/usb.dd hash=md5,sha1 hashlog=/tmp/hashlog.txt
```

![dcfldd hash generation](/assets/images/mirai/135-9.png)
*Imaging the device while computing hashes*

The image is then verified by comparing the hashes:

```bash
dcfldd if=/dev/sdb vf=/tmp/usb.dd verifylog=/tmp/verifylog.txt
```

![Hash verification](/assets/images/mirai/135-10.png)
*Verifying the image against the source device*

![Exfiltrating image to local machine](/assets/images/mirai/135-11.png)
*Transferring the image to the attacking machine*

![USB image ready for forensic tools](/assets/images/mirai/135-12.png)
*The image staged for analysis*

## File Extraction & Root Flag

**fls** lists the files and directories inside the image, which exposes the deleted **root.txt** flag. Running **strings** over the image extracts the printable content and recovers the flag value.

![Strings command output with root.txt](/assets/images/mirai/135-13.png)
*Recovering the root flag from the image with strings*
