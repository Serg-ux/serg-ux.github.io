## Reconnaissance

The engagement began with a full TCP port scan to identify the services exposed on the target.

```bash
nmap -sVC -p- --open -sS --min-rate 5000 -v -n -Pn -oN grandpa_scan <IP>
```

Port 80 is open, serving an HTTP service identified as Microsoft IIS httpd 6.0. Because that version is badly outdated, a `searchsploit` lookup is run, which returns several known vulnerabilities affecting it.

![searchsploit results for IIS 6.0](/assets/images/grandpa/132-1.png)
*Known exploits returned by searchsploit for IIS 6.0*

## Exploitation

Metasploit ships a module that exploits Microsoft IIS 6.0 through a remote buffer overflow, so that module is selected and run against the target. Exploitation succeeds and returns a Meterpreter session.

![Meterpreter session obtained](/assets/images/grandpa/132-2.png)
*Initial Meterpreter session on the target*

Local user enumeration shows the Harry and Administrator accounts, although neither can be accessed directly at this stage.

## Privilege Escalation

With no immediate escalation path, the Metasploit local exploit suggester is run to find applicable vulnerabilities.

![Local exploit suggester output](/assets/images/grandpa/132-3.png)
*Local exploit suggester results*

It reports several valid local exploits. Before using them, the Meterpreter session is migrated into a process running as NT AUTHORITY\NETWORK SERVICE to obtain a stable context.

![Suggested local exploits](/assets/images/grandpa/132-4.png)
*Candidate local privilege-escalation modules*

From the suggestions, the following module is chosen.

```bash
exploit/windows/local/ms14_070_tcpip_ioctl
```

Running it elevates the session and grants administrative privileges on the target.

![Elevated session with admin privileges](/assets/images/grandpa/132-5.png)
*Session elevated to administrative privileges*
